# Subagent Base: I18n Translation Worker

> 本文件包含所有翻译子代理共享的通用规则。主 agent 委派时，必须将本文件内容连同目标语言参数和差异文件清单一并注入。

主 agent 将提供以下参数，请严格使用：

- **TARGET_LANG**：目标语言名（`English` 或 `日本語`）
- **OUTPUT_DIR**：输出目录（`docs/en/` 或 `docs/ja/`）
- **DATE_LABEL**：日期行格式
  - English: `> 📅 Last Updated:`
  - 日本語: `> 📅 最終更新日:`
- **FILES**：需要处理的文件清单（路径 + 操作类型：新建 / 更新 / 删除）

---

## 路径映射规则

目标文档路径由源路径直接替换语言根目录得到：

```
docs/zh-CN/README.md           →  {OUTPUT_DIR}/README.md
docs/zh-CN/src/__init__.md     →  {OUTPUT_DIR}/src/__init__.md
docs/zh-CN/src/runtime/foo.md  →  {OUTPUT_DIR}/src/runtime/foo.md
docs/zh-CN/tests/bar.md        →  {OUTPUT_DIR}/tests/bar.md
```

> `docs/zh-CN/` 下的任意 `.md` 文件，只要存在镜像路径，都纳入同步范围。

### 操作规则

- **新建**：读取 `docs/zh-CN/` 对应文件全文，翻译后写入 `{OUTPUT_DIR}/` 镜像路径。若父目录不存在则先创建。
- **更新**：读取 `docs/zh-CN/` 对应文件全文，重新翻译并**覆盖**写入 `{OUTPUT_DIR}/` 镜像路径（不是增量 diff，是全量重译以保持结构一致）。
- **删除**：将 `{OUTPUT_DIR}/` 对应文件删除（视为已废弃内容）。
- **跳过**：不做任何操作。

---

## 翻译规则

### 核心原则

- **zh-CN 是唯一事实来源**：翻译时不得添加 zh-CN 中没有的信息，也不得删减 zh-CN 中已有的信息。
- **忠实优先于优美**：优先保证技术含义准确，其次才是目标语言的自然流畅。
- **不必要则不修改**：对于已有的翻译内容，如果对应部分的 zh-CN 原文没有变化，则**原样保留**现有译文，不要为提高表达质量或调整措辞而重译。只在 zh-CN 实际变更的部分做增量翻译。
- **术语一致性**：同一技术术语在整份文档中的译法应保持一致。建议首次出现时在译文后附原文（如 English: "TaskGraph (任务图)"，日本語: "TaskGraph（タスクグラフ）"）。

### 内容分类处理

翻译时按内容类别区别对待：

| 内容类型 | 处理方式 |
|----------|---------|
| 中文正文段落 | **翻译**为目标语言 |
| 中文标题 (`#`, `##`, `###` 等) | **翻译**为目标语言 |
| 代码块 (` ```python `, ` ```bash ` 等) | **原样保留**（含代码注释） |
| Mermaid 图表 (` ```mermaid `) | **原样保留**节点 ID 和结构，仅翻译节点标签中的中文文本和 `subgraph` 标题 |
| HTML 标签（`<img>`, `<a>`, `<p>` 等） | **原样保留** |
| URL / 文件路径 | **原样保留** |
| 内联代码 (`` `code` ``) | **原样保留** |
| Markdown 表格 | **翻译**表头及单元格文字内容，保持列对齐 |
| 引用块 (`>`) | **翻译**引用内容（但日期行用 `{DATE_LABEL}` 本地化） |
| 列表项 (`-`, `1.`) | **翻译**列表文字 |

### 日期本地化

处理文档开头的日期行时：

- 如果 zh-CN 有日期行（`> 📅 最后更新日期: YYYY/MM/DD`）→ 在译文相同位置写入 `{DATE_LABEL} YYYY/MM/DD`，日期值取 zh-CN 的值。
- 如果 zh-CN 无日期行 → 译文也不写日期行。
- 如果目标文件已有旧日期行但格式不对 → 修正为上述格式，日期值同步为 zh-CN 的值。

### Mermaid 翻译细则

仅翻译**面向读者的文本**，保留所有结构定义：

- ✅ **翻译**：节点标签中的中文（如 `Graph[任务图]` → EN: `Graph[Task Graph]` / JA: `Graph[タスクグラフ]`）、`subgraph` 标题中的中文
- ❌ **保留**：节点 ID（如 `A`, `B`, `Graph`, `Queue`）、连线类型（`-->`, `--->`）、Mermaid 关键字（`flowchart`, `subgraph`, `end`）、连线标签中的非中文内容

示例：

```
原文：
flowchart LR
    A[开始] --> B{检查}
    B -->|是| C[执行]

en 译文：
flowchart LR
    A[Start] --> B{Check}
    B -->|Yes| C[Execute]

ja 译文：
flowchart LR
    A[開始] --> B{チェック}
    B -->|はい| C[実行]
```

### 标题编号与锚点

- 保留编号格式不变（如 `1.`、`1.1`）
- 仅翻译编号后的文字
- Markdown 锚点（`[text](#anchor)`）中的 `#anchor` 部分**原样保留**，仅翻译显示文本

---

## 翻译质量要求

- 翻译结果应自然流畅，符合目标语言表达习惯。
- 技术术语优先使用业界通用译法。
- 长段落可适当拆分，避免译文出现"翻译腔"过重的超长句。
- 代码示例中的注释如果本身是中文，**不翻译**（注释面向开发者阅读源码时使用，翻译反而造成干扰）。
- 如果 zh-CN 中已经混有英文术语或英文句子，**原样保留**不翻译。

---

## 输出格式（统一要求）

完成后**必须**按以下格式输出报告：

```markdown
## 语言: {TARGET_LANG}

### 新建文档 (N 个)
| 源文件 (zh-CN) | 目标文件 |
|---------------|---------|
| `docs/zh-CN/...` | `{OUTPUT_DIR}/...` |
| ... | |

### 更新文档 (N 个)
| 源文件 (zh-CN) | 目标文件 | 更新原因 |
|---------------|---------|---------|
| `docs/zh-CN/...` | `{OUTPUT_DIR}/...` | 日期过期 / 内容缺失 |
| ... | | |

### 删除文档 (N 个)
| 已删除文件 | 原因 |
|-----------|------|
| `{OUTPUT_DIR}/...` | zh-CN 中已不存在 |
| ... | |

### 跳过文档 (N 个)
| 文件 | 原因 |
|------|------|
| `{OUTPUT_DIR}/...` | 日期一致，无需更新 |
| ... | |

### ⚠️ 发现的问题
- （如 zh-CN 中疑似错误、Mermaid 图表中未翻译的遗留文本、无法处理的特殊情况等）
```
