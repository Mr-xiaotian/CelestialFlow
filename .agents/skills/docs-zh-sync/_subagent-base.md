# Subagent Base Rules

> 本文件包含所有子代理共享的通用规则。主 agent 委派子代理时，必须将本文件内容与对应区域的 `subagent-*.md` 一并注入消息。

## 路径映射规则

### 根目录映射

| 代码路径 | 文档路径 |
|---------|---------|
| `src/celestialflow/...` | `docs/zh-CN/src/...` |
| `bench/...` | `docs/zh-CN/bench/...` |
| `tests/...` | `docs/zh-CN/tests/...` |
| `demo/...` | `docs/zh-CN/demo/...` |

### 后缀映射

| 代码后缀 | 文档后缀 |
|:-------:|:-------:|
| `.py` | `.md` |
| `.ts` | `.md` |
| `.html` | `.md` |
| `.css` | `.md` |
| `__init__.py` | `__init__.md` |

### 示例

| 代码文件 | 文档文件 |
|---------|---------|
| `src/celestialflow/runtime/util_errors.py` | `docs/zh-CN/src/runtime/util_errors.md` |
| `src/celestialflow/web/static/ts/main.ts` | `docs/zh-CN/src/web/static/ts/main.md` |
| `src/celestialflow/web/templates/index.html` | `docs/zh-CN/src/web/templates/index.md` |
| `src/celestialflow/web/static/css/dashboard.css` | `docs/zh-CN/src/web/static/css/dashboard.md` |
| `tests/runtime/test_queue.py` | `docs/zh-CN/tests/runtime/test_queue.md` |
| `demo/demo_graph.py` | `docs/zh-CN/demo/demo_graph.md` |

如果项目里已经存在旧版但不镜像的中文文档路径，优先以"镜像路径"作为目标；必要时说明发现了旧路径遗留问题。

## 审计清单（按优先级逐项排查）

每次审计时，按以下频率表从高到低优先排查：

| 频率 | 模式 | 典型表现 | 排查方法 |
|:----:|------|---------|----------|
| 🔴 极高 | **虚构功能** | 文档声称支持的功能/方法/类，源码无对应实现 | 对照源码每个公开方法逐一确认 |
| 🔴 极高 | **遗漏新增** | 源码重构后新增了类/函数/异常，文档完全未提及 | `grep "class \|def "` 对照文档中的函数列表 |
| 🔴 极高 | **重构后旧名残留** | 函数/类/参数/配置项改名后，文档仍用旧名 | `grep` 源码中的 `def`/`class`/`__init__` 签名与文档名称交叉比对 |
| 🔴 极高 | **模型/配置结构重构未同步** | Pydantic 从字段模式改为 `RootModel`，配置从平铺改为嵌套分组 | 关注 model 类的继承变化、配置字段层级变化 |
| 🟠 高 | **参数名过期** | `current_name` → `out_name`，`max_retries` → 实际不存在 | 关注源码 `__init__` 签名 vs 文档中的参数表 |
| 🟠 高 | **函数归属错误** | 函数实际在 `main.ts` 中，文档错误记在 `utils.ts` 下 | 用 `grep` 确认函数定义位置 |
| 🟠 高 | **CSS 类名/选择器漂移** | 文档描述的 CSS 类名（如 `.sortable-dragging`）源码中已删除 | `grep` CSS 文件中的类选择器 vs 文档 |
| 🟠 高 | **文件拆分/合并未同步** | 源码将一个文件拆成多个或合并，但文档结构未跟随调整 | `find_path` 对比代码/文档文件数量 |
| 🟡 中 | **API 端点虚构** | 文档列了 `/api/pull_history_limit`，源码中无此路由 | `grep` 源码中的路由注册或 URL 常量 |
| 🟡 中 | **算法/性能解释错误** | 用"GIL 竞争"解释耗时差异，实际是算法复杂度 O(φⁿ) vs O(n) | 并行读取相关源文件，理解实际算法 |
| 🟡 中 | **`__all__` 列表过期** | `__init__.py` 新增导出但 `__init__.md` 未列出 | 读取 `__init__.py` 末尾的 `__all__` 与文档对比 |
| 🟢 低 | **旧路径遗留** | 文档在 `tests/` 根目录下平铺，但源码已迁移到子目录 | 按镜像规则全量核对路径 |
| 🟢 低 | **路径不镜像** | 文档路径不符合镜像规则 | 同上 |

此外也不应忽略：接口签名过期、示例与当前代码不一致、仍在描述已删除能力。

## 文档写作规则（通用）

### 基本原则

- **以代码为唯一事实来源**，不虚构功能。必要时结合同目录已有文档风格。
- **默认使用中文**。
- **重点写**：职责、结构、关键函数、关键类、数据流、异常、配置、注意事项。
- **不要仅把函数名罗列出来而没有解释**。
- **不要遗漏最近的重构结果**，尤其是重命名、异常类型变化、字段变化、状态键变化。
- 如果发现文档内容与代码冲突，**按代码修正文档**，并在最终总结里指出冲突点。

### 标注规则（极其重要）

> **不要轻易删除原有内容。** 原文档中已有的信息允许修改、补充、重组，但除非该信息已被代码证实为完全错误或彻底废弃，否则不要删除。宁可保留旧信息并标注，也不要让有价值的历史说明丢失。

标注写法示例：

```
> ⚠️ **已变更**：原文档描述的 `injection.css` 已拆分为 4 个独立文件，详见各子文档。

> ⚠️ **已废弃**：`loadSummary()` 函数已在 vX.Y 中移除，汇总信息现由前端聚合计算。

> ⚠️ **待确认**：以下信息来自旧版文档，当前源码中未找到对应实现，保留供参考。
```

### 日期更新规则

- 更新/新建文档的 `最后更新日期` 设为当前日期（`YYYY/MM/DD` 格式）。
- **未修改的文档保持原有日期不变。**

### 三类文件的特殊写法

- **`__init__.py` → `__init__.md`**：列出模块全部公开导出符号，按子模块分组。包根入口还应有完整的 `__all__` 列表和包导入使用示例。
- **`util_*.py` → `util_*.md`**：逐函数说明用途、参数、返回值和使用示例。函数数量较少时逐函数列表，数量多时表格化。
- **`conftest.py` → `conftest.md`**：说明提供的 fixture 及其用途，不需要运行方式。

### 使用示例要求

- **完整可运行是底线**：每个 Python/TS 文档的示例必须包含必需的 `import` 语句。
- **一个示例胜过十行描述**：优先用带语言标注的代码块展示用法。
- **覆盖典型路径**：至少展示该模块最常见的 2-3 种使用场景。

## 文档表达形式

善用丰富的表达形式，原则：**形式服务于内容**。

| 形式 | 适用场景 | 用法要点 |
|------|---------|---------|
| `flowchart` (Mermaid) | 数据流、架构拓扑、处理步骤 | 默认 `LR` 或 `TD`；用 `subgraph` 对节点分组 |
| `sequenceDiagram` | API 交互、模块间调用顺序 | 保留 `loop`/`alt` 展示重复和条件分支 |
| `classDiagram` | 核心类关系和继承结构 | 存在基类和多个子类时优先使用 |
| `stateDiagram-v2` | 生命周期和状态机 | 适合 Spout、Executor 等有明确状态转换的组件 |
| 表格 | 对比、配置项罗列、参数说明 | 结构化信息比纯文字更清晰 |
| 代码块 | 关键示例、运行命令 | 带语言标注：\`\`\`python、\`\`\`ts、\`\`\`bash |
| 引用块 `>` | 重要提示、注意事项 | 突出设计决策背景和风险提示 |

**节点命名技巧**：英文主名 + `<br/>` 中文简注，如 `LogSpout<br/>日志消费者`。

**文档骨架建议**（可按文件类型灵活调整）：

```md
# 文件或模块标题

> 最后更新日期: YYYY/MM/DD

## 作用

## 核心对象 / 核心函数

## 关键流程

## 重要细节

## 使用示例 / 测试重点 / 运行方式

## 注意事项
```
