# bench_hash_memory.py 基准测试说明

> 📅 最后更新日期: 2026/06/16

## 目标

对比 SHA1 哈希值以 `str`（40 字符 hex）、`bytes`（20 字节原始）、`int`（160-bit 整数）三种类型存入 `set` 时的内存占用与查询性能，为 `processed_set` 的数据类型选择提供依据。

## 测试策略

| 类型 | 构造方式 | 单对象大小 | 说明 |
|------|----------|-----------|------|
| `str` | `randbytes(20).hex()` | 81 B | 原始方案，40 字符 hex 字符串 |
| `bytes` | `randbytes(20)` | 53 B | 当前方案，`hashlib.sha1().digest()` 直接输出 |
| `int` | `int.from_bytes(randbytes(20), 'big')` | 48 B | 最紧凑，但失去固定长度语义 |

## 测量维度

- **单对象大小**：`sys.getsizeof()` 返回的 Python 对象内存
- **set 总内存增量**：`tracemalloc` 现场构造 N 个对象加入 set，快照差值
- **每条目平均开销**：总增量 / N（含 set 桶、指针等开销）
- **构建耗时**：N 次 `set.add()` 的总时间
- **查询延迟**：命中 / 未命中各 0.3s 稳态测量，报告单次纳秒数

## 基准结果（实测）

### 历史结果 - Windows 11 N=100,000（时间未记录）

> 环境：Windows 11，Python 3.14，N=100,000

| 类型 | 单对象(B) | 总内存(MB) | 每条(B) | 构建(ms) | Hit(ns) | Miss(ns) |
|------|----------|-----------|---------|---------|---------|----------|
| `str` | 81 | 11.73 | 123.0 | 60.40 | 110.5 | 112.2 |
| `bytes` | 53 | 9.06 | 95.0 | 56.23 | 112.7 | 112.7 |
| `int` | 48 | 8.58 | 90.0 | 69.05 | 121.9 | 112.4 |

#### 内存节省（相对 str）

| 类型 | 总内存 | 每条开销 |
|------|--------|---------|
| `bytes` | 77.2% | 77.2% |
| `int` | 73.2% | 73.2% |

**关键结论**：
- `str→bytes` 节省约 23% 内存，百万级任务下约省 27 MB
- `bytes→int` 仅再省约 5%，但 `int` 无固定长度语义，调试不直观
- 三者查询性能无显著差异（~112 ns），类型切换不影响去重效率
- **推荐使用 `bytes`**：`hashlib.sha1().digest()` 直接返回，零转换开销

### 2026/06/16 - 本地复测

> 环境：Windows，N=100,000

| 类型 | 单对象(B) | 总内存(MB) | 每条(B) | 构建(ms) | Hit(ns) | Miss(ns) |
|------|----------|-----------|---------|---------|---------|----------|
| `str` | 81 | 11.73 | 123.0 | 62.83 | 70.8 | 69.9 |
| `bytes` | 53 | 9.06 | 95.0 | 37.33 | 74.5 | 71.5 |
| `int` | 48 | 8.58 | 90.0 | 47.45 | 76.3 | 70.0 |

#### 内存节省（相对 str）

| 类型 | 总内存 | 每条开销 |
|------|--------|---------|
| `bytes` | 77.2% | 77.2% |
| `int` | 73.2% | 73.2% |

**本轮补充结论**：
- `bytes` 仍然是最均衡的方案：内存显著优于 `str`，构建速度也最快
- `int` 在内存上略优于 `bytes`，但构建速度反而更慢，查询也没有优势
- 本轮查询延迟整体落在 `70-76 ns`，三种格式差异很小，去重查找不是类型选择的决定因素

## 参数调整

### 修改测试规模

在 `bench/bench_hash_memory.py` 的顶部修改 `N` 值：

```python
N = 10_000          # 小规模快速验证
# N = 1_000_000     # 大规模测试（观察百万级内存差异）
```

### 只测试特定类型

脚本中的类型对比循环在 `for name, factory in [("str", make_str), ...]` 中定义，可通过注释跳过部分类型：

```python
for name, factory in [
    # ("str", make_str),   # 跳过 str 类型
    ("bytes", make_bytes),  # 仅测试 bytes
    # ("int", make_int),    # 跳过 int 类型
]:
```

### 自定义随机种子

```python
random.seed(123)  # 修改种子以获得不同哈希值分布
```

修改后运行：

```bash
python bench/bench_hash_memory.py
```
