# **Kahn 算法**（卡恩算法）

核心思想是：

> **不断剥离入度为 0 的节点，如果最终能剥离完全部节点，则图中无环；否则必有环。**

---

## 1. 为什么入度为 0 的节点一定安全？

入度为 0 意味着**没有任何边指向它**。在有向图中，这样的节点不可能是任何环的一部分（因为环要求每个节点至少被一条边指向）。

所以我们可以放心地把它（以及它发出的所有边）从图中删除，不会影响剩余图中是否存在环。

---

## 2. 算法步骤拆解

以这段代码为例：

```python
def is_dag(graph: OrderGraph) -> bool:
    deg = in_degree(graph)              # ① 计算所有节点入度
    stack = [n for n, d in deg.items() 
             if d == 0]                 # ② 收集所有入度为0的节点
    visited = 0                          # ③ 计数：已被剥离的节点数

    while stack:                         # ④ 不断处理入度为0的节点
        u = stack.pop()                  # ⑤ 取出一个节点
        visited += 1                     # ⑥ 计数+1

        for v in graph._out.get(u, []):  # ⑦ 遍历它的所有出边 u→v
            deg[v] -= 1                  # ⑧ 删除这条边：v的入度-1
            if deg[v] == 0:              # ⑨ 如果v的入度恰好变为0
                stack.append(v)          # ⑩ 说明v现在"安全"了，加入处理队列

    return visited == len(graph._nodes) # ⑪ 全部剥离完？是则DAG
```

### 逐行解释

| 步骤 | 动作 | 意义 |
|------|------|------|
| **①** | `deg = in_degree(graph)` | 先算每个节点被多少条边指着。例如 `a←b` 则 `a` 的入度 +1。 |
| **②** | `stack = [入度为0的节点]` | 这些节点是当前图的"起点"，不可能是环的一部分。 |
| **③** | `visited = 0` | 记录我们已经"剥离"了多少个节点。 |
| **④~⑥** | `while stack: pop → visited += 1` | 每次从栈里拿出一个安全节点，宣布它已被移除。 |
| **⑦~⑧** | `for v in out[u]: deg[v] -= 1` | **关键**：节点 `u` 被移除后，它发出的所有边也要跟着消失。边消失意味着下游节点 `v` 少了一条入边，所以 `v` 的入度减 1。 |
| **⑨~⑩** | `if deg[v] == 0: stack.append(v)` | **核心**：当 `v` 的最后一条入边被删光后，`v` 变成了新的"起点"，此时它也不可能再属于任何环，可以安全加入处理队列。 |
| **⑪** | `visited == len(nodes)` | 如果所有节点都被剥离，说明图可以被完全拓扑排序，**无环**；如果 `visited < len(nodes)`，说明剩下一些节点怎么也剥不掉——它们互相指来指去，构成了**环**。 |

## 3. 直观例子

假设图如下：

```
a → b → c
↑       ↓
└───────┘   (c 指向 a，形成环)
```

**初始入度：**

| 节点 | 入度 | 来源 |
|------|------|------|
| a    | 1    | ← c |
| b    | 1    | ← a |
| c    | 1    | ← b |

**执行过程：**

1. 找入度为 0 的节点：`[]`（没有！）
2. `stack` 为空，`while` 循环直接结束。
3. `visited = 0`，`len(nodes) = 3`。
4. `0 == 3`？**否** → `is_dag` 返回 `False`。

**为什么？** 因为 a、b、c 的入度都是 1，谁都不是起点，谁也删不掉，它们构成了环。

---

再假设图是 DAG：

```
a → b → c
```

**初始入度：** a=0, b=1, c=1

**第 1 轮：** `stack = [a]`  
- 弹出 `a`，`visited = 1`  
- 删除 `a→b`，`b` 入度变为 0，入栈  
- `stack = [b]`

**第 2 轮：**  
- 弹出 `b`，`visited = 2`  
- 删除 `b→c`，`c` 入度变为 0，入栈  
- `stack = [c]`

**第 3 轮：**  
- 弹出 `c`，`visited = 3`  
- `c` 无出边，无后续操作

结束：`visited == 3` → **是 DAG**。

---

## 4. 为什么环中的节点永远进不了 stack？

假设节点 `x` 属于某个环。环中的每个节点至少被环内的另一条边指着，所以它们的入度至少为 1（仅来自环内的部分）。

当你剥离环外的节点时，可能会减少一些入度，但**环内部的边永远不会被删除**（因为环内的节点谁也进不了 stack，谁也删不掉，它们发出的边也就永远不会被删除）。

因此，环内节点的入度**永远不可能降到 0**，它们永远卡在 `deg >= 1` 的状态，最终 `visited` 无法覆盖它们。

---

## 5. 复杂度

- **时间：** $O(V + E)$。每个节点最多入栈出栈一次，每条边最多被"删除"一次（触发一次 `deg[v] -= 1`）。
- **空间：** $O(V)$。栈和入度数组的大小。

---

## 总结

`is_dag()` 的本质是**模拟拓扑排序**：

> 能排完序（所有节点都被剥离）= 无环 = DAG  
> 排不完（剩下一坨互相指着的节点）= 有环 = 不是 DAG

这也是为什么在代码里 `topo_sort()` 和 `is_dag()` 长得几乎一模一样——`is_dag` 其实就是拓扑排序的"判定版"，只关心能不能排完，不记录顺序。
