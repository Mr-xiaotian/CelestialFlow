# i18n.ts

> 📅 Last Updated: 2026/06/22

Internationalization module, responsible for maintaining multi-language (Chinese, English, Japanese) translation data, current language state management, and automatic DOM text updates.

## Type Definitions

### `Lang`
Supported language identifiers: `"zh-CN"` | `"en"` | `"ja"`.

## Global Variables

| Variable | Type | Description |
|------|------|------|
| `currentLang` | `Lang` | Currently selected language identifier, defaults to `"zh-CN"` |
| `translations` | `Record<Lang, Record<string, string>>` | Translation dictionary containing all languages |

## Core Functions

### `setLang(lang)`

Switches the global language setting.

- **Parameter**: `lang: Lang` - target language identifier.
- **Behavior**: Updates `currentLang` and synchronously modifies the `lang` attribute of the `<html>` element.

---

### `t(key, ...args)`

Gets the translated text for the current language based on a translation key, supporting simple placeholder replacement.

- **Parameters**:
  - `key: string` - translation key name.
  - `...args: string[]` - argument list for replacing `{0}`, `{1}`...
- **Returns**: The translated string. Returns the key name itself if no matching key is found.
- **Fallback mechanism**: If the key is not found in the current language, falls back to `zh-CN`.

---

### `applyI18nDOM()`

Scans the entire page for elements with specific `data-i18n` attributes and applies translations.

**Supported attributes:**
- `data-i18n`: Replaces `textContent`.
- `data-i18n-placeholder`: Replaces `placeholder`.
- `data-i18n-title`: Replaces `title` attribute.
- `data-i18n-aria-label`: Replaces `aria-label` attribute.

Additionally, this function synchronously updates `document.title`.

## Translation Key Groups

- `app.*`: Application global text (title, etc.)
- `settings.*`: Settings panel related
- `tab.*`: Tab names
- `card.*`: Card titles
- `summary.*`: Summary data labels
- `errors.*`: Error log related
- `injection.*`: Task injection related
- `status.*`: Node status related
- `chart.*`: Chart related
- `interval.*`: Time interval options

## Usage Example

### t() / setLang() / applyI18nDOM() Usage Example

The following example demonstrates the usage of the three core functions of the i18n module in the browser console:

```typescript
// 1. t(): Get the translated text for the current language
console.log(t("app.title"));       // "Task Graph Monitoring System" (current en)
console.log(t("settings.saveSuccess")); // "Settings saved"

// 2. t() supports placeholder replacement
// Translation key "injection.successBatch": "Successfully submitted drafts for {0} nodes"
console.log(t("injection.successBatch", "3"));
// "Successfully submitted drafts for 3 nodes"

// 3. t() fallback mechanism
// If the current language (e.g. ja) doesn't have a key, falls back to zh-CN
// If zh-CN also doesn't have it, returns the key name itself
console.log(t("nonexistent.key"));  // "nonexistent.key"

// 4. setLang(): Switch the global language
setLang("en");
console.log(currentLang);           // "en"
console.log(document.documentElement.lang); // "en"
console.log(t("app.title"));       // "Task Graph Monitor"

setLang("ja");
console.log(t("app.title"));       // "タスクグラフモニター"

// 5. applyI18nDOM(): Scan the entire page and update translations
// Assuming the HTML has:
//   <h1 data-i18n="app.title"></h1>
//   <input data-i18n-placeholder="errors.searchPlaceholder">
//   <div data-i18n-title="settings.tooltip"></div>
//
// After calling applyI18nDOM():
//   <h1>Task Graph Monitoring System</h1>
//   <input placeholder="Search errors or tasks">
//   <div title="Settings"></div>

// 6. Complete language switching flow
function switchLanguage(lang: Lang) {
    setLang(lang);                    // Update currentLang and <html lang>
    applyI18nDOM();                   // Update all data-i18n elements
    document.title = t("app.title"); // Update page title
}

// Switch to English
switchLanguage("en");

// Switch to Chinese
switchLanguage("zh-CN");

// 7. Check if a translation key exists
function hasTranslation(key: string, lang?: Lang): boolean {
    const targetLang = lang || currentLang;
    return !!translations[targetLang]?.[key];
}

console.log(hasTranslation("app.title"));                 // true
console.log(hasTranslation("app.title", "en"));           // true
console.log(hasTranslation("app.title", "ja"));           // true
console.log(hasTranslation("nonexistent.key", "zh-CN"));  // false
```
