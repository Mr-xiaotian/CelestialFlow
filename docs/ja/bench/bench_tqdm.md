# bench_tqdm.py ベンチマーク説明

> 📅 最終更新日: 2026/06/16

## 目的

`tqdm` プログレスバーのループ内におけるパフォーマンスオーバーヘッドを定量化し、超大規模反復（数百万レベル）でプログレス表示を有効にすべきかの判断を支援する。

## テスト内容

- **tqdm あり**：各反復で `pbar.update(1)` を呼び出し
- **tqdm なし**：ベアループ
- **データ規模**：関数シグネチャのデフォルトは `data_size = 10_000_000`、ただし `__main__` 内の実際の呼び出しは `1_000_000` を使用
- **シミュレーション処理**：`item * 2`（極めて軽量な演算）

## 主要パラメータ

- `dynamic_ncols=True`：ターミナル幅に自動適応
- `total=0` の後、動的に `pbar.total = len(data)` を設定：合計数の遅延設定の使用法をデモ

## 発生し得る問題

1. **TTY 検出オーバーヘッド**：出力がファイルやパイプにリダイレクトされた場合、`tqdm` は表示を自動無効化する可能性があるが、一部のリフレッシュロジックは依然として実行され、直接ターミナル実行時と結果が異なる場合がある。
2. **`dynamic_ncols` のターミナルクエリ**：各リフレッシュでターミナル幅をクエリする。特定の CI 環境や Windows PowerShell では遅いシステムコールをトリガーする可能性がある。
3. **大規模ループでのメモリ増加**：テストコードは `range(data_size)` を完全にリスト展開して `data` に格納する。`data_size` を 1000 万に引き上げた場合、リストだけで約 80MB のメモリを消費する。

## ベンチマーク結果（実測）

### 履歴結果 - Windows 軽量ループ（日時未記録）

> 環境：Windows、Python 3.10、data_size=1,000,000、処理ロジックは `item * 2`

| モード | 準備時間 | 処理時間 | 合計時間 | tqdm なし比 |
|------|----------|----------|--------|-------------|
| **tqdm なし** | 0.0859s | 0.0334s | **0.1194s** | — |
| **tqdm あり** | 0.1067s | 0.3259s | **0.4325s** | **3.6x** |

**主要な結論**：
- tqdm は本テストで合計所要時間を約 **3.6x** 増加させた（0.12s → 0.43s）
- 処理時間が 0.033s から 0.326s に増加し、約 10x の増加。**極めて軽量なループ本体**では、tqdm のリフレッシュオーバーヘッドが実際の計算を大きく上回る
- 単一反復の計算量が大きい場合（> 1ms 程度）、tqdm の相対オーバーヘッドは急速に無視できるレベル（< 5%）まで低下する
- 戦略提案：100 万レベルの軽量ループでは tqdm をオフにするか `miniters` を増やす。長時間/少数回タスクでは安心して有効化できる

### 2026/06/16 - ローカル再テスト

> 環境：Windows、`data_size=1,000,000`、処理ロジックは `item * 2`

| モード | 準備時間 | 処理時間 | 合計時間 | tqdm なし比 |
|------|----------|----------|--------|-------------|
| **tqdm なし** | 0.0436s | 0.0207s | **0.0644s** | — |
| **tqdm あり** | 0.0679s | 0.1933s | **0.2611s** | **4.1x** |

**今回の補足結論**：
- 現在のマシンでは `tqdm` が合計所要時間を約 **4.1x** 増加させ、依然としてプログレスバーなし版より明らかに高い
- 処理フェーズが `0.0207s` から `0.1933s` に増加しており、軽量ループでの主要コストは引き続きリフレッシュと端末出力に由来する
- 履歴結果と比較して絶対時間は全体的に低下しているが、「軽量タスクでは tqdm がオーバーヘッドを明らかに増幅する」という結論に変化はない

## 実行方法

```bash
python bench/bench_tqdm.py
```

## パラメータ調整

### データ規模の変更

`test_tqdm_performance(data_size)` は `data_size` パラメータを受け付ける：

```bash
# 異なる規模の比較を実行
python -c "
from bench.bench_tqdm import test_tqdm_performance
test_tqdm_performance(use_tqdm=False, data_size=100_000)
test_tqdm_performance(use_tqdm=True, data_size=100_000)
"
```

`if __name__ == "__main__"` 内で直接変更も可能：

```python
if __name__ == "__main__":
    # 小規模テスト（素早く検証）
    test_tqdm_performance(use_tqdm=False, data_size=10_000)
    test_tqdm_performance(use_tqdm=True, data_size=10_000)

    # 大規模テスト（tqdm オーバーヘッド比率の変化を観察）
    # test_tqdm_performance(use_tqdm=False, data_size=10_000_000)
    # test_tqdm_performance(use_tqdm=True, data_size=10_000_000)
```

修正後に実行：

```bash
python bench/bench_tqdm.py
```
