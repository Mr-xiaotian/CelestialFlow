# bench_hash_container.py ベンチマーク説明

> 📅 最終更新日: 2026/05/13

## 目的

`bytes` をハッシュ型として使用することが決定した後、20 バイト SHA1 ハッシュを格納する異なるコンテナ構造のメモリオーバーヘッドとクエリ性能を比較し、`processed_set` のコンテナ選定（純粋な set vs. エビクション付き LRU）にデータを提供する。

## テストコンテナ

| コンテナ | 説明 | 適用シナリオ |
|----------|------|-------------|
| `set[bytes]` | 基準、最もコンパクト | エビクション不要、タスク量が制御可能 |
| `dict[bytes, None]` | dict の最小オーバーヘッド | dict インターフェースが必要だが値は不要 |
| `dict[bytes, float]` | タイムスタンプを格納 | 時間ベースの期限切れエントリのエビクション |
| `OrderedDict[bytes, None]` | 順序付き dict | LRU の基本構造 |
| LRU(unlimited) | OrderedDict + `move_to_end` | LRU セマンティクス、容量制限なし |
| LRU(50k) | 同上、容量上限 50,000 | メモリ制約シナリオ、最も古い未使用をエビクション |

## 測定次元

- **コンテナ合計メモリ増分**：`tracemalloc` スナップショット差分
- **エントリあたりの平均オーバーヘッド**：合計増分 / エントリ数
- **構築時間**：N 回の挿入（LRU の `move_to_end` / `popitem` オーバーヘッド含む）
- **クエリ遅延**：ヒット / ミス各 0.3 秒の定常状態測定

## ベンチマーク結果（実測）

> 環境：Windows 11、Python 3.14、N=100,000

| コンテナ | エントリ数 | 合計メモリ(MB) | エントリあたり(B) | 構築(ms) | Hit(ns) | Miss(ns) |
|----------|-----------|---------------|------------------|---------|---------|----------|
| `set[bytes]` | 100,000 | 4.00 | 42.0 | 8.52 | 112.7 | 112.0 |
| `dict[B,None]` | 100,000 | 5.00 | 52.4 | 8.87 | 113.9 | 112.7 |
| `dict[B,float]` | 100,000 | 7.29 | 76.4 | 65.29 | 113.8 | 111.9 |
| `OrderedDict` | 100,000 | 10.05 | 105.4 | 42.08 | 122.5 | 119.4 |
| LRU(unlimited) | 100,000 | 10.05 | 105.4 | 57.25 | 115.6 | 115.1 |
| LRU(50k) | 50,000 | 8.53 | 178.8 | 69.95 | 124.0 | 115.3 |

### メモリ比較（set[bytes] 基準）

| コンテナ | 倍率 | 絶対値 |
|----------|------|--------|
| `dict[B,None]` | 125% | 5.00 MB |
| `dict[B,float]` | 182% | 7.29 MB |
| `OrderedDict` | 251% | 10.05 MB |
| LRU(unlimited) | 251% | 10.05 MB |
| LRU(50k) | 213% | 8.53 MB |

**主要な結論**：
- `set[bytes]` が最もコンパクト（42 B/エントリ）だが、古いエントリをエビクションできず、長期実行ではメモリが線形増加
- `dict[B,None]` はメモリが 25% 多いだけ。バッチクリーンアップのみが必要な場合（例：バッチごとの `clear()`）は最良の選択
- `OrderedDict` / LRU のメモリは set の 2.5 倍（双方向リンクリストのオーバーヘッド）だが、O(1) エビクション機能を提供
- LRU(50k) のメモリはタスク量に関係なく最大約 8.5 MB。代償としてウィンドウ外の重複タスクの検出漏れが発生
- すべてのコンテナのクエリ性能は近い（112-124 ns）。コンテナの選択はスループットに影響しない
- **推奨**：デフォルトでは `set[bytes]` を使用。メモリ上限が必要な場合は `OrderedDict` ベースの LRU を使い、`maxsize` はビジネス許容度に応じて設定
