# bench_hash_memory.py ベンチマーク説明

> 📅 最終更新日: 2026/06/11

## 目的

SHA1 ハッシュ値を `str`（40 文字 hex）、`bytes`（20 バイト生データ）、`int`（160-bit 整数）の 3 つの型で `set` に格納した場合のメモリ使用量と検索性能を比較し、`processed_set` のデータ型選定に根拠を提供する。

## テスト戦略

| 型 | 構築方法 | 単一オブジェクトサイズ | 説明 |
|----|---------|----------------------|------|
| `str` | `randbytes(20).hex()` | 81 B | 旧方式、40 文字 hex 文字列 |
| `bytes` | `randbytes(20)` | 53 B | 現行方式、`hashlib.sha1().digest()` の直接出力 |
| `int` | `int.from_bytes(randbytes(20), 'big')` | 48 B | 最もコンパクト、ただし固定長セマンティクスを喪失 |

## 測定軸

- **単一オブジェクトサイズ**：`sys.getsizeof()` が返す Python オブジェクトメモリ
- **set 総メモリ増分**：`tracemalloc` で N 個のオブジェクトをその場で構築し set に追加、スナップショット差分
- **エントリあたり平均オーバーヘッド**：総増分 / N（set のバケット、ポインタ等のオーバーヘッドを含む）
- **構築所要時間**：N 回の `set.add()` の合計時間
- **検索レイテンシ**：ヒット / ミス各 0.3s 定常測定、単一回あたりのナノ秒数を報告

## ベンチマーク結果（実測）

> 環境：Windows 11、Python 3.14、N=100,000

| 型 | 単一オブジェクト(B) | 総メモリ(MB) | エントリ毎(B) | 構築(ms) | Hit(ns) | Miss(ns) |
|----|--------------------|-------------|--------------|---------|---------|----------|
| `str` | 81 | 11.73 | 123.0 | 60.40 | 110.5 | 112.2 |
| `bytes` | 53 | 9.06 | 95.0 | 56.23 | 112.7 | 112.7 |
| `int` | 48 | 8.58 | 90.0 | 69.05 | 121.9 | 112.4 |

### メモリ節約（str 比）

| 型 | 総メモリ | エントリ毎オーバーヘッド |
|----|---------|------------------------|
| `bytes` | 77.2% | 77.2% |
| `int` | 73.2% | 73.2% |

**主要な結論**：
- `str→bytes` で約 23% のメモリ節約、100 万タスク規模で約 27 MB 節約
- `bytes→int` でさらに約 5% 節約、ただし `int` は固定長セマンティクスがなく、デバッグが直感的でない
- 3 者間で検索性能に有意差なし（~112 ns）、型の切り替えは重複排除効率に影響しない
- **`bytes` の使用を推奨**：`hashlib.sha1().digest()` が直接返す値で、変換オーバーヘッドがゼロ

## 実行方法

```bash
python bench/bench_hash_memory.py
```

## パラメータ調整

### テスト規模の変更

`bench/bench_hash_memory.py` の先頭で `N` 値を変更：

```python
N = 10_000          # 小規模で素早く検証
# N = 1_000_000     # 大規模テスト（100 万件レベルのメモリ差を観察）
```

### 特定型のみをテスト

スクリプト内の型比較ループは `for name, factory in [("str", make_str), ...]` で定義されており、コメントアウトで一部の型をスキップできる：

```python
for name, factory in [
    # ("str", make_str),   # str 型をスキップ
    ("bytes", make_bytes),  # bytes のみテスト
    # ("int", make_int),    # int 型をスキップ
]:
```

### カスタム乱数シード

```python
random.seed(123)  # シードを変更して異なるハッシュ値分布を得る
```

修正後に実行：

```bash
python bench/bench_hash_memory.py
```
