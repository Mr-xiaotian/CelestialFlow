# 更新履歴（Change Log）

> 📅 最終更新日: 2026/06/05

- 2021: マルチスレッドとシングルスレッドの処理関数をサポートするクラスを確立しました
- 2023: GPT-4 の支援により、マルチプロセスとコルーチンの実行モードを追加しました
- 5/9/2024: 既存の処理クラスをノードに抽象化し、`TaskChain` クラスを追加しました。複数のノードを線形に接続し、Chain 内の各ノードの実行モードを設定できます。`"serial"` と `"process"` の2種類をサポートし、後者は Chain 内の全ノードを同時に実行します
- 12/12/2024-12/16/2024: 既存のチェーン構造を基に、ノードが複数の下流ノードを持てるようにし、Tree 構造を実現しました。従来の `TaskChain` を `TaskTree` に改名しました
- 3/16/2025: Web 上でのタスク完了状況の可視化をサポートしました
- 6/9/2025: ノードが複数の上流ノードを持てるようになり、純粋な Tree 構造から脱却し、その後の循環グラフへの準備を整えました
- 6/11/2025: [CelestialVault](https://github.com/Mr-xiaotian/CelestialVault) プロジェクトの `instances.inst_task` から移行しました
- 6/12/2025: 循環グラフをサポートしました。下流ノードが上流ノードを指すことができます
- 6/13/2025: loop 構造をサポートしました。ノードが自分自身を指すことができます
- 6/14/2025: forest 構造をサポートしました。複数のルートノードを持つことができます
- 6/16/2025: 複数回の評価を経て、現在のフレームワークは完全な有向グラフ構造をサポートするようになり、`TaskTree` を `TaskGraph` に改名しました
- 3.0.1: PyPI に公開しました。喜ばしい限りです
- 3.0.4: ノードのすべての「入辺」と「出辺」を表す新しい抽象構造 `TaskQueue` を追加しました。未消費タスクの保存機能を復元しました
- 3.0.5: 既存の `TaskRedisTransfer` ノードを削除し、3種類の新しい Redis 連携ノード `TaskRedisTransport`、`TaskRedisSource`、`TaskRedisAck` を追加しました。クロス言語・クロスプロセス・クロスデバイスのタスク処理に対応します。Web ページにトポロジー情報を表示するカードを追加しました
- 3.0.6: [CelestialTree](https://github.com/Mr-xiaotian/CelestialTree) システムのサポートを追加しました。個々のタスクのフローを追跡できるようになりました
- 3.0.7: `TaskStage` を `TaskExecutor` から独立したサブクラスとして抽出しました。新しいノード `TaskRouter` を追加し、受信タスクをブロードキャストではなく、選択的に異なる下流ノードにルーティングできるようになりました
- 3.0.8: ctree のロジックにおいて、「タスクリトライ」イベント後の「タスク成功/失敗/リトライ」イベントを、以前の並列関係ではなく因果関係として扱うようにしました。エラー収集ロジックをリファクタリングしました。バージョン 3.0.6 と 3.0.7 で導入された多数のバグを修正し、一部のログ出力を改善しました
