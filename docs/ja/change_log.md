# 更新履歴（Change Log）

> 📅 最終更新日: 2026/05/28

- 2021: マルチスレッドとシングルスレッドの処理関数をサポートするクラスを作成しました
- 2023: GPT-4の支援により、マルチプロセスとコルーチンの実行モードを追加しました
- 5/9/2024: 既存の処理クラスをノードに抽象化し、TaskChainクラスを追加しました。複数のノードを線形に接続し、Chain内の各ノードの実行モードを設定できます。`serial`と`process`の2種類をサポートし、後者はChain内の全ノードを同時に実行します
- 12/12/2024-12/16/2024: 既存のチェーン構造を拡張し、ノードが複数の下流ノードを持てるようにしてTree構造を実現しました。TaskChainをTaskTreeに改名しました
- 3/16/2025: Web上でのタスク完了状況の可視化をサポートしました
- 6/9/2025: ノードが複数の上流ノードを持てるようになり、純粋なTree構造から脱却し、循環グラフへの準備を整えました
- 6/11/2025: [CelestialVault](https://github.com/Mr-xiaotian/CelestialVault)プロジェクトのinstances.inst_taskから分離しました
- 6/12/2025: 循環グラフをサポートしました。下流ノードが上流ノードを指すことができます
- 6/13/2025: ループ構造をサポートしました。ノードが自分自身を指すことができます
- 6/14/2025: フォレスト構造をサポートしました。複数のルートノードを持つことができます
- 6/16/2025: 複数回の評価の結果、フレームワークが完全な有向グラフ構造をサポートするようになり、TaskTreeをTaskGraphに改名しました
- 3.0.1: PyPIに公開しました。めでたいことです
- 3.0.4: ノードのすべての「入辺」と「出辺」を表す新しい抽象構造TaskQueueを追加しました。未消費タスクの保存機能を復元しました
- 3.0.5: 既存のTaskRedisTransferノードを削除し、3種類の新しいRedis連携ノード — TaskRedisTransport、TaskRedisSource、TaskRedisAck — を追加しました。クロス言語・クロスプロセス・クロスデバイスのタスク処理に対応します。Webページにトポロジー情報カードを追加しました
- 3.0.6: [CelestialTree](https://github.com/Mr-xiaotian/CelestialTree)システムのサポートを追加しました。個々のタスクのフローを追跡できるようになりました
- 3.0.7: TaskStageをTaskExecutorから独立したサブクラスとして抽出しました。新しいノードTaskRouterを追加し、受信タスクをブロードキャストではなく、選択的に異なる下流ノードにルーティングできるようになりました
- 3.0.8: CelestialTreeのロジックにおいて、「タスクリトライ」イベント後の「タスク成功/失敗/リトライ」イベントを、以前の並列関係ではなく因果関係として扱うようにしました。エラー収集ロジックをリファクタリングし、バージョン3.0.6と3.0.7で導入された多数のバグを修正し、一部のログ出力を改善しました
