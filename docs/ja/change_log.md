# 更新ログ（Change Log）

> 📅 最終更新日: 2026/06/18

- 2021: マルチスレッドとシングルスレッド処理関数をサポートするクラスを構築
- 2023: GPT-4 の支援でマルチプロセスとコルーチン実行モードを追加
- 5/9/2024: 既存の処理クラスをノードに抽象化し、`TaskChain` クラスを追加。複数のノードを線形に接続し、Chain 内のノード実行モードを設定可能。`"serial"` と `"process"` をサポート。後者は Chain の全ノードが同時実行
- 12/12/2024-12/16/2024: 既存のチェーン構造を基に、ノードが複数の下位ノードを持てるようにし、Tree 構造を実現。既存の `TaskChain` を `TaskTree` に改名
- 3/16/2025: Web 側でのタスク完了状況の可視化をサポート
- 6/9/2025: ノードが複数の上位ノードを持てるようにし、純粋な Tree 構造から脱却。後の循環グラフへの準備
- 6/11/2025: [CelestialVault](https://github.com/Mr-xiaotian/CelestialVault) プロジェクトの `instances.inst_task` から移行
- 6/12/2025: 循環グラフをサポート。下位ノードが上位ノードを指せるように
- 6/13/2025: loop 構造をサポート。ノードが自分自身を指せるように
- 6/14/2025: forest 構造をサポート。複数のルートノードを持てるように
- 6/16/2025: 複数回の評価を経て、現在のフレームワークが完全な有向グラフ構造をサポート。`TaskTree` を `TaskGraph` に改名
- 3.0.1: PyPI に公開、可喜可賀
- 3.0.4: 新しい抽象構造 `TaskQueue` を追加し、ノードのすべての「入辺」と「出辺」を表現。未消費タスクの保存機能を復元
- 3.0.5: 既存の `TaskRedisTransfer` ノードを削除し、3 種類の新しい Redis 対話ノード `TaskRedisTransport`、`TaskRedisSource`、`TaskRedisAck` を追加。クロス言語、クロスプロセス、クロスデバイスのタスク処理に使用。Web ページにトポロジー情報を表示するカードを追加
- 3.0.6: [CelestialTree](https://github.com/Mr-xiaotian/CelestialTree) システムのサポートを追加。単一タスクのフローを追跡可能に
- 3.0.7: `TaskStage` を `TaskExecutor` から独立したサブクラスとして抽出。新しいノード `TaskRouter` を追加。受信したタスクを選択的に異なる下流ノードに転送可能（ブロードキャストではなく）
- 3.0.8: ctree のロジック上で「タスクリトライ」イベント後の「タスク成功/失敗/リトライ」イベントを因果関係として扱うよう変更（以前は並行関係）。エラー収集ロジックの一部をリファクタリング。3.0.6 と 3.0.7 バージョンで導入された多数のバグを修正。一部の log 表現を最適化
