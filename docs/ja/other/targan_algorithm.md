# **Tarjan 算法**

有向グラフの**すべての強連結成分（SCC）**を $O(V+E)$ の時間で求める。これはグラフ論の中でも最も巧妙なアルゴリズムの一つであり、核心は 3 つの要素だけである：**DFS 時間戳**、**lowlink 値**、**スタックの維持**。

---

## 1. 強連結成分（SCC）とは？

**強連結**：ノード $u$ と $v$ が互いに到達可能であること（$u \to v$ かつ $v \to u$）。

**強連結成分**：任意の 2 点が強連結である極大な部分グラフ。言い換えれば、SCC 内部のノードは互いに行き来できるが、外部とは「一方通行の壁」になっている——SCC から出ることはでき、外部から入ることもできるが、一度出たら戻ってこれない（別の SCC 間を除く）。

**重要な定理**：各 SCC を 1 つの点に縮約すると、得られる**凝聚图（Condensation Graph）は必ず有向无环图（DAG）**となる。

---

## 2. Tarjan 算法の核心思想

Tarjan は DFS の過程で 3 つのことを行う：

| メカニズム | 役割 |
|------------|------|
| **`indices[v]`（dfn）** | ノード $v$ が DFS で初めて訪問された**時間戳**（増加する整数） |
| **`lowlink[v]`** | $v$ から出発し、**树边 + 回边 + スタック内ノードを指す横叉边**を通じて到達できる最小の時間戳 |
| **栈 `stack`** | 現在の DFS パス上で**まだ SCC 所属が確定していない**ノードを維持する |

### 核心判定条件

```
if lowlink[v] == indices[v]:
    v 是一个 SCC 的"根"
    从栈顶不断弹出，直到 v 为止，这些节点构成一个 SCC
```

**なぜか？** `lowlink[v]` は $v$ が「たどり着ける」最も古い祖先を表す。もしそれが自分自身にしかたどり着けない場合（`lowlink[v] == indices[v]`）、$v$ を根とする部分木内に、$v$ の部分木より前の祖先に飛び出せるノードが存在しないことを示す。したがって、$v$ の部分木内でまだスタックに残っているすべてのノードは、$v$ と一緒に閉じた SCC を形成する。

---

## 3. コードを逐行分解する

```python
def tarjan_scc(graph: OrderGraph) -> list[list[str]]:
    out = graph._out
    index = 0                    # 全局时间戳计数器
    stack: list[str] = []        # 维护当前"活跃"节点
    on_stack: set[str] = set()   # 快速判断某节点是否在栈中
    indices: dict[str, int] = {} # 记录每个节点的发现时间戳
    lowlink: dict[str, int] = {} # 记录每个节点能回溯到的最小时间戳
    sccs: list[list[str]] = []   # 结果：所有 SCC 列表
```

### 内側関数 `strongconnect(v)`

```python
    def strongconnect(v: str) -> None:
        nonlocal index
        
        # ① 初始化：给 v 打上时间戳，lowlink 先等于自己
        indices[v] = lowlink[v] = index
        index += 1
        stack.append(v)
        on_stack.add(v)
```

**第一步**：ノード $v$ が発見される。`indices[v]` と `lowlink[v]` は最初、現在の時間戳に等しくなる。$v$ はスタックに入り、「まだ組織を見つけていない」ことを示す。

```python
        # ② 遍历所有出边 v → w
        for w in out.get(v, []):
            
            if w not in indices:
                # 情况 A：w 未被访问过 → 树边，继续 DFS
                strongconnect(w)
                # 回溯后，w 的 lowlink 已经算好，用它来更新 v
                lowlink[v] = min(lowlink[v], lowlink[w])
            
            elif w in on_stack:
                # 情况 B：w 在栈中 → 回边或指向同 SCC 的横叉边
                # w 的时间戳就是它能代表的最老祖先
                lowlink[v] = min(lowlink[v], indices[w])
            
            # 情况 C（隐式 else）：w 已访问但不在栈中
            # 说明 w 属于一个已经确定的 SCC，对 v 无影响，直接忽略
```

**3 種類の辺タイプの処理**：

| ケース | 条件 | 意味 | 操作 |
|--------|------|------|------|
| **A** | `w not in indices` | 树边（Tree Edge）、$w$ は $v$ の子ノード | 再帰 DFS を行い、回溯後に `lowlink[w]` を使って `lowlink[v]` を更新する |
| **B** | `w in on_stack` | 回边（Back Edge）、$w$ は $v$ の祖先か同じ SCC 内にある | `indices[w]` を使って `lowlink[v]` を更新し、$v$ が $w$ に戻れることを示す |
| **C** | `w in indices` かつ `w not in on_stack` | 横叉边/前向边、**已完成した SCC** を指す | 無視する。その SCC はすでに閉じられており、$v$ はそれを通じて新しい強連結を形成できない |

```python
        # ③ 判定：v 是不是某个 SCC 的根？
        if lowlink[v] == indices[v]:
            scc = []
            while True:
                w = stack.pop()
                on_stack.remove(w)
                scc.append(w)
                if w == v:
                    break
            sccs.append(scc)
```

**第三步**：$v$ のすべての出边を DFS し終えた後、`lowlink[v] == indices[v]` であれば、$v$ の部分木内に $v$ より前に飛び出せるノードがないことを示す。このとき、スタックの先頭から $v$ までのすべてのノードが、ちょうど 1 つの完全な SCC を構成するため、それらをすべてポップする。

### 外側ループ

```python
    for v in graph._nodes:
        if v not in indices:
            strongconnect(v)
    
    return sccs
```

グラフは連結でない可能性があるため、すべてのノードを走査し、未訪問のノードから DFS を開始する。

## 4. 完整な例を追う

以下のグラフ構造を仮定する：

```
a → b → c
↑       ↓
└───────┘      (a,b,c 形成环: SCC0)

c → d → e
        ↑
        └──── f  (e,f 形成环: SCC1)
```

邻接表：
- `a: [b]`
- `b: [c]`
- `c: [a, d]`
- `d: [e]`
- `e: [f]`
- `f: [e]`

### 実行過程

**a を訪問：**
- `indices[a]=0, lowlink[a]=0`, スタック `[a]`

**a→b：**
- `b` は未訪問なので再帰。
- `indices[b]=1, lowlink[b]=1`, スタック `[a, b]`

**b→c：**
- `c` は未訪問なので再帰。
- `indices[c]=2, lowlink[c]=2`, スタック `[a, b, c]`

**c→a：**
- `a` は已訪問かつ `a in on_stack`（ケース B）
- `lowlink[c] = min(2, 0) = 0`

**c→d：**
- `d` は未訪問なので再帰。
- `indices[d]=3, lowlink[d]=3`, スタック `[a, b, c, d]`

**d→e：**
- `e` は未訪問なので再帰。
- `indices[e]=4, lowlink[e]=4`, スタック `[a, b, c, d, e]`

**e→f：**
- `f` は未訪問なので再帰。
- `indices[f]=5, lowlink[f]=5`, スタック `[a, b, c, d, e, f]`

**f→e：**
- `e` は已訪問かつ `e in on_stack`（ケース B）
- `lowlink[f] = min(5, 4) = 4`

**f の DFS 終了：**
- `lowlink[f]=4 != indices[f]=5` のため、ポップしない。返回。
- `e` を更新：`lowlink[e] = min(4, 4) = 4`

**e の DFS 終了：**
- `lowlink[e]=4 != indices[e]=4`？**待て、等しい！**
- そう、`lowlink[e] == indices[e] == 4`
- **SCC をポップ：** スタックの先頭から `f`、`e` を順にポップ。`['f', 'e']` を得る（ポップ順であり、逆順にすると `[e, f]`）
- スタックは `[a, b, c, d]` に
- `sccs = [['f', 'e']]`（この SCC は汇点 SCC で、最初に見つかる）

**d に戻る：**
- `lowlink[d] = min(3, 4) = 3` を更新

**d の DFS 終了：**
- `lowlink[d]=3 == indices[d]=3`
- **SCC をポップ：** `d` をポップ。`['d']` を得る
- スタックは `[a, b, c]` に
- `sccs = [['f', 'e'], ['d']]`

**c に戻る：**
- `lowlink[c] = min(0, 3) = 0` を更新

**c の DFS 終了：**
- `lowlink[c]=0 != indices[c]=2` のため、ポップしない。

**b に戻る：**
- `lowlink[b] = min(1, 0) = 0` を更新

**b の DFS 終了：**
- `lowlink[b]=0 != indices[b]=1` のため、ポップしない。

**a に戻る：**
- `lowlink[a] = min(0, 0) = 0` を更新

**a の DFS 終了：**
- `lowlink[a]=0 == indices[a]=0`
- **SCC をポップ：** `c, b, a` を順にポップ。`['c', 'b', 'a']` を得る
- スタックは空になる
- `sccs = [['f', 'e'], ['d'], ['c', 'b', 'a']]`

### 最終結果

```python
[['f', 'e'], ['d'], ['c', 'b', 'a']]
```

**順序に注意**：これは**逆トポロジカル順**——**汇点 SCC が先に出力され**、**源点 SCC が後に出力される**。

対応する凝聚図：
```
SCC(a,b,c) → SCC(d) → SCC(e,f)
```

Tarjan の出力順：`[SCC(e,f), SCC(d), SCC(a,b,c)]` で、ちょうどトポロジカルソートの**逆順**である。

---

## 5. なぜ返回顺序は「逆トポロジカル順」なのか？

Tarjan 算法では、ある SCC はその**すべての出边が DFS 完了**した後でしかポップされない。つまり：

- SCC A から SCC B へ辺がある場合、DFS は先に B へ深く入り、B が先にポップされ、A は後にポップされる。
- したがって、先にポップされた SCC は凝聚図において「下流」（汇点方向）、後にポップされたものは「上流」（源点方向）にある。

これはまさに凝聚図のトポロジカルソートの**逆順**である。この性質は非常に有用であり、後続で `source_sccs` を探す際、最後のいくつかの SCC の中から入度 0 のものを探せばよい。

---

## 6. 計算量分析

- **時間**：$O(V + E)$。各ノードを 1 回訪問し、各辺を 1 回チェックし、スタック操作は償却 $O(1)$。
- **空間**：$O(V)$。スタック、時間戳配列、再帰スタックの深さは最大で $V$。

---

## 7. よくある誤解

| 誤解 | 説明 |
|------|------|
| "なぜケース C（已訪問だがスタックにない）を無視するのか？" | $w$ はすでに確定した SCC に属しており、その SCC は凝聚図で $v$ の下流にある。$v$ は $w$ を通じてより古い祖先に戻れないため、`lowlink` には影響しない。 |
| "ポップされた SCC の内部順序に意味はあるのか？" | ない。`['c', 'b', 'a']` は単なるポップ順であり、トポロジー関係を表さない。SCC 内部では互いに到達可能である。 |
| "lowlink の更新でなぜ `lowlink[w]` ではなく `indices[w]` を使うのか？" | ケース B（回边）では、$w$ はまだスタックにあり、$w$ と $v$ は同じ DFS 木内の未閉鎖領域に属する。`indices[w]` は $w$ としての祖先の「高さ」を表すため、それを直接使用して更新すればよい。 |

---

## まとめ

Tarjan 算法の精髓は一言でまとめられる：

> **DFS 時間戳を使ってノードに番号を付け、lowlink を使って遡れる最も古い祖先を追跡し、あるノードが自分自身にしか遡れないことに気づいたとき、それが一塊の強連結成分の"根"であり、スタック内のその塊をすべて吐き出せば、それが 1 つの SCC となる。**

Kahn 算法の `is_dag` と組み合わせることで、両者は完璧な相補関係を形成する：Tarjan が**环の分解**を担当し、Kahn が**无环の検証**を担当する。
