# **Kahn 算法**（カーン算法）

核心的な考え方は：

> **入度が 0 のノードを次々に剥がしていき、最終的にすべてのノードを剥がせればグラフにサイクルはなく、そうでなければ必ずサイクルが存在する。**

---

## 1. 入度が 0 のノードがなぜ安全か？

入度が 0 とは、**自分に向かう辺が一切ない**ことを意味する。有向グラフにおいて、このようなノードは任意のサイクルの一部になり得ない（サイクルでは各ノードは少なくとも 1 本の辺によって指されなければならないため）。

したがって、このノード（およびそこから出るすべての辺）をグラフから削除しても、残りのグラフにサイクルが存在するかどうかには影響しない。

---

## 2. アルゴリズムのステップ分解

以下のコードを例に説明する：

```python
def is_dag(graph: OrderGraph) -> bool:
    deg = in_degree(graph)              # ① 计算所有节点入度
    stack = [n for n, d in deg.items() 
             if d == 0]                 # ② 收集所有入度为0的节点
    visited = 0                          # ③ 计数：已被剥离的节点数

    while stack:                         # ④ 不断处理入度为0的节点
        u = stack.pop()                  # ⑤ 取出一个节点
        visited += 1                     # ⑥ 计数+1

        for v in graph._out.get(u, []):  # ⑦ 遍历它的所有出边 u→v
            deg[v] -= 1                  # ⑧ 删除这条边：v的入度-1
            if deg[v] == 0:              # ⑨ 如果v的入度恰好变为0
                stack.append(v)          # ⑩ 说明v现在"安全"了，加入处理队列

    return visited == len(graph._nodes) # ⑪ 全部剥离完？是则DAG
```

### 行ごとの解釈

| ステップ | 動作 | 意味 |
|----------|------|------|
| **①** | `deg = in_degree(graph)` | 各ノードが何本の辺で指されているかを先に計算する。例えば `a←b` なら `a` の入度が +1 される。 |
| **②** | `stack = [入度为0的节点]` | これらのノードは現在のグラフの「起点」であり、サイクルの一部になり得ない。 |
| **③** | `visited = 0` | すでに「剥がした」ノード数を記録する。 |
| **④~⑥** | `while stack: pop → visited += 1` | 毎回、stack から安全なノードを 1 つ取り出し、削除済みとしてカウントする。 |
| **⑦~⑧** | `for v in out[u]: deg[v] -= 1` | **要点**：ノード `u` が削除されると、そこから出るすべての辺も消える。辺が消えることで下流ノード `v` は 1 本の入辺を失うため、`v` の入度を 1 減らす。 |
| **⑨~⑩** | `if deg[v] == 0: stack.append(v)` | **核心**：`v` の最後の入辺が削除され、`v` が新しい「起点」になったとき、それもサイクルに属し得ないため、安全に処理待ち stack に追加できる。 |
| **⑪** | `visited == len(nodes)` | すべてのノードが剥がせれば、グラフを完全にトポロジカルソートできたことになり、**サイクルなし**；`visited < len(nodes)` なら、どうやっても剥がれないノードが残っている——つまり互いに指し合って**サイクル**を形成している。 |

---

## 3. 具体的な例

以下のグラフを仮定する：

```
a → b → c
↑       ↓
└───────┘   (c 指向 a，形成环)
```

**初期入度：**

| ノード | 入度 | 来源 |
|--------|------|------|
| a      | 1    | ← c |
| b      | 1    | ← a |
| c      | 1    | ← b |

**実行過程：**

1. 入度 0 のノードを探す：`[]`（なし！）
2. `stack` が空なので、`while` ループはすぐ終了する。
3. `visited = 0`、`len(nodes) = 3`。
4. `0 == 3`？**いいえ** → `is_dag` は `False` を返す。

**なぜか？** a、b、c の入度はすべて 1 で、誰も起点ではなく、誰も削除できない。つまり、それらはサイクルを形成している。

---

DAG であると仮定する：

```
a → b → c
```

**初期入度：** a=0, b=1, c=1

**ラウンド 1：** `stack = [a]`  
- `a` をポップ、`visited = 1`  
- `a→b` を削除し、`b` の入度が 0 になったので stack に追加  
- `stack = [b]`

**ラウンド 2：**  
- `b` をポップ、`visited = 2`  
- `b→c` を削除し、`c` の入度が 0 になったので stack に追加  
- `stack = [c]`

**ラウンド 3：**  
- `c` をポップ、`visited = 3`  
- `c` からの出辺はないため、それ以上の処理はない

終了：`visited == 3` → **DAG である**。

---

## 4. なぜサイクル内のノードは永遠に stack に入れないのか？

ノード `x` があるサイクルに属していると仮定する。サイクル内の各ノードは、サイクル内の別の辺によって少なくとも 1 本指されているため、それらの入度は少なくとも 1 である（サイクル内部分のみで考えても）。

サイクル外のノードを剥がすとき、一部の入度は減るかもしれないが、**サイクル内部の辺は永遠に削除されない**（サイクル内のノードは誰も stack に入れず、誰も削除できないため、そこから出る辺も永遠に削除されない）。

したがって、サイクル内ノードの入度は**永遠に 0 まで下がらず**、最終的に `visited` はそれらをカバーできない。

---

## 5. 計算量

- **時間：** $O(V + E)$。各ノードは最大 1 回 stack に入り出るし、各辺は最大 1 回「削除」される（`deg[v] -= 1` が 1 回実行される）。
- **空間：** $O(V)$。stack と入度配列のサイズ。

---

## まとめ

`is_dag()` の本质は**トポロジカルソートをシミュレート**することである：

> すべてのノードを剥がせた（ソートが完了した）= サイクルなし = DAG  
> ソートが完了しなかった（互いに指し合うノードの塊が残った）= サイクルあり = DAG でない

これが、コード内で `topo_sort()` と `is_dag()` がほとんど同じように見える理由でもある——`is_dag` はトポロジカルソートの「判定版」であり、ソートが完了するかどうかだけを気にし、順序自体は記録しない。
