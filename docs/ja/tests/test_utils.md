# test_utils.py テストユーティリティ説明

> 📅 最終更新日: 2026/04/22

## テスト目的

`test_utils.py` はテストスイートの共有ユーティリティライブラリであり、単体テストとデモテストに統一されたテスト関数、データジェネレーター、ヘルパークラスを提供します。すべてのテストタスク関数は**純粋関数または制御された副作用関数**であり、予測可能で再現性のあるテスト動作を保証します。

## テスト範囲

### 1. 汎用計算関数
フレームワークのコア計算ロジックを検証するための数学関数：

| 関数 | 説明 | 特性 |
|------|------|------|
| `fibonacci(n)` | 再帰フィボナッチ | `n <= 0` の場合 `ValueError` を発生、例外リトライのテストに使用 |
| `fibonacci_async(n)` | 非同期再帰フィボナッチ | `async` 実行モードの検証 |
| `no_op(n)` | 恒等関数 | 計算オーバーヘッドなし、ベンチマーク対照として使用 |
| `sum_int(*num)` | 整数合計 | 複数引数のアンパック検証（`unpack_task_args=True`） |
| `add_one(num)` | 1を加算 | 最も基本的な線形変換 |
| `sqrt(num)` | 平方根 | 浮動小数点演算の検証 |
| `square(x)` | 二乗 | `sleep(1)` の遅延を含む、時間のかかるタスクのシミュレーション |

### 2. 例外境界付き演算関数

| 関数 | トリガー条件 | 例外タイプ |
|------|------------|-----------|
| `add_offset(x, offset=10)` | `x > 30` | `ValueError` |
| `add_5` / `add_10` / `add_15` など | 同上（オフセット値が異なる） | `ValueError` |

これらの関数は以下の検証に使用されます：
- 例外のキャプチャと統計
- リトライメカニズム（`ValueError` に対するリトライを設定）
- 部分的な失敗シナリオにおけるデータフローの継続性

### 3. スリープバリアント

| 関数 | 説明 |
|------|------|
| `sleep_1(n)` | 同期1秒遅延 |
| `sleep_1_async(n)` | 非同期1秒遅延 |
| `add_one_sleep(n)` | 1秒遅延後に1を加算、複数条件の例外あり |

### 4. URL 処理関数（demo_stages 専用）

Web クローラーワークフローをシミュレートする関数群：
- `generate_urls_sleep` / `log_urls_sleep` / `download_sleep` / `parse_sleep`
- 各関数には4〜6秒のランダム遅延が含まれ、I/O 集約型タスクをシミュレートします
- `download_to_file` は `requests` を使用して実際の HTTP リクエストを実行します

### 5. 特殊クラス

#### `RouterWrapper`
`TaskRouter` テスト用のルーティングラッパー：
- 偶数は `a_tag` にルーティング
- 奇数は `b_tag` にルーティング
- フレームワークのリフレクション要件を満たすため `__name__` 属性を設定する必要があります

## 発生しうる問題と注意事項

### 1. `sleep` を含む関数によるテストの遅延
`square`、`add_offset`、`sleep_1` などの関数にはハードコードされた `sleep(1)` またはランダムな4〜6秒の遅延が含まれています。単体テストではこれらの関数を直接使用せず、`add_one` や `double` などの遅延のない関数を使用してください。

**解決策**：
- 単体テストファイル（例：`test_executor.py`）では内部に独立した高速関数を定義
- デモテスト（例：`demo_structure.py`）では実際のワークロードをシミュレートするため遅延関数を使用

### 2. `fibonacci` の再帰の深さ
`fibonacci(32)` は既に約200万回の再帰呼び出しを含んでおり、以下を引き起こす可能性があります：
- Python の再帰深度制限（デフォルト1000）
- テスト実行時間の過剰な長さ

**推奨事項**：単体テストでは `n <= 10` の小さな値のみを使用してください。

### 3. `download_to_file` のネットワーク依存
この関数は実際の HTTP リクエストを発行するため、以下のリスクがあります：
- 対象 URL が無効になりテストが失敗する
- ネットワークの変動によるタイムアウト
- ローカルファイルシステムの権限問題（Windows では `/tmp/` パスが存在しない場合がある）

**推奨事項**：デモ環境でのみ使用してください。単体テストでは `responses` または `httpx.MockTransport` を使用してモックしてください。

### 4. `random` の非決定性
`generate_urls` は `random.randint(1, 4)` を使用してランダムな数の URL を生成するため、以下が発生します：
- 毎回の実行でタスク数が異なる
- 成功/失敗数の正確なアサーションが困難

**推奨事項**：正確なアサーションが必要な場合は、テスト前にランダムシードを設定してください：`random.seed(42)`。

## 実行方法

このファイル自体にはテストケースは含まれておらず、インポートされる共有モジュールとして使用されます：
```python
from tests.test_utils import add_one, fibonacci
```
