# ハッシュユーティリティテスト (test_hash.py)

> 📅 最終更新日: 2026/05/28

## 目的

`celestialflow.runtime.util_hash` 内の `make_hashable` と `object_to_hash` の 2 つのハッシュユーティリティ関数を検証し、任意の Python オブジェクトが安定してハッシュ可能な形式に変換され、一意のダイジェストを生成できることを確認します。

## コアテスト対象

- `make_hashable`: リスト、辞書、集合などのハッシュ不可能なオブジェクトを再帰的にハッシュ可能な構造（タプル）に変換します。
- `object_to_hash`: 任意のオブジェクトに対して 20 バイトの SHA1 ダイジェストを計算し、タスクの重複排除に使用します。

## 主要テストシナリオ

### `make_hashable`
- **基本型はそのまま返却**：`int`、`str`、`float`、`bool`、`None` は変更なし
- **タプルはそのまま返却**：それ自体がハッシュ可能で、変換不要
- **空コンテナ**：`[]` → `()`、`{}` → `()`
- **リストからタプル**：`[1, 2, 3]` → `(1, 2, 3)`
- **ネストされたリスト**：`[[1, 2], [3, 4]]` → `((1, 2), (3, 4))`
- **辞書**：決定性を保証するため、ソートされた `(key, value)` タプルペアに変換
- **ネストされた辞書**：内部の値も再帰的に変換
- **混合型**：辞書と基本型を含むリスト
- **集合**：ソートされたタプルに変換
- **結果がハッシュ可能**：変換結果は `set` 要素 / `dict` キーとして使用可能

### `object_to_hash`
- **戻り値の型**：20 バイトの `bytes`
- **冪等性**：同じオブジェクトを複数回呼び出しても同じハッシュを生成
- **同構造同ハッシュ**：構造的に同一のオブジェクトは同じハッシュを生成
- **異値異ハッシュ**：`1`、`"1"`、`[1]`、`{"a": 1}` はすべて異なる
- **ネスト構造**：複雑なネストオブジェクトも正しく計算
- **空コンテナの区別**：`[]`、`{}`、`()` の 3 つの空コンテナは異なるハッシュを生成

## 実行方法

```bash
# 全実行
pytest tests/utils/test_hash.py -v

# make_hashable テストのみ
pytest tests/utils/test_hash.py -k "make_hashable" -v

# object_to_hash テストのみ
pytest tests/utils/test_hash.py -k "object_to_hash" -v

# ネスト構造テストのみ

```

## 重要な詳細

- `make_hashable` は辞書と集合をキーでソートしてから変換し、`{"b": 2, "a": 1}` と `{"a": 1, "b": 2}` が同じ結果を生成することを保証します。
- `object_to_hash` は内部的に `make_hashable` を呼び出し、その後 `pickle` でシリアル化して SHA1 を計算します。
- 両方のハッシュメカニズムはフレームワークの重複排除に使用されます：`make_hashable` は高速なメモリ内比較用、`object_to_hash` は永続化 / プロセス間比較用です。

## 注意事項

- ハッシュユーティリティはタスク重複排除機能の基盤であり、誤った実装は重複タスクの見逃しや誤検出につながります。
- 関連実装は `src/celestialflow/runtime/util_hash.py` にあります。
