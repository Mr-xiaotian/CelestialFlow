# フォーマットユーティリティテスト (test_format.py)

> 📅 最終更新日: 2026/06/22

## 役割
`celestialflow.utils.util_format` の汎用フォーマットツールを検証し、端末出力、ログ記録、レポート生成時にデータが見やすく美しい方法で表示されることを確認します。

## コアテスト対象
- `format_repr`: インテリジェントな文字列短縮とエスケープツール。
- `format_table`: 文字端末テーブル生成ツール。

## 主要テストフロー
1. **文字列短縮 (format_repr)**:
   - 短い文字列がそのまま保持されることを検証。
   - 長い文字列が `前半...後半` 形式に切り詰められ、合計長が期待通りであることを検証。
   - 特殊文字（改行、バックスラッシュ）が正しくエスケープされ、ログ形式が破壊されないことを検証。
2. **テーブルフォーマット (format_table)**:
   - **基本機能**: 2 次元リストが枠線付き文字テーブルに変換され、行・列インデックスが自動生成されることを検証。
   - **カスタム設定**: カスタム行名、列名、異なる配置（left/right）のレンダリング効果を検証。
   - **耐障害性**: 不規則なネストリスト（列数が一致しない）が `fill_value` で埋めて整列され、空データが親切なプロンプトを返すことを検証。

## テストの重点
- **境界整列**: 異なる長さの内容でもテーブル枠線が整列していることを確認。
- **エスケープ安全性**: `\n` などの文字が repr モードで実際の改行を生成しないことを確認。
- **メモリ安全性**: テストケースは小規模データを使用し、フォーマットロジックが再帰や複雑なループによってパフォーマンス問題を引き起こさないことを確認。

## 実行方法

```bash
# すべて実行
pytest tests/utils/test_format.py -v

# 文字列短縮テストのみ実行
pytest tests/utils/test_format.py -k "repr" -v

# テーブルフォーマットテストのみ実行
pytest tests/utils/test_format.py -k "table" -v
```

## 重要な詳細
- テーブルフォーマットアルゴリズムは各列の最大幅を自動計算します。
- `test_format_repr_truncation` は切り詰めロジックの比率分布を検証します。

## 注意事項
- これらのツールは主に CLI ツールチェーンと内部デバッグログに使用されます。
- 関連実装は `src/celestialflow/utils/util_format.py` にあります。
