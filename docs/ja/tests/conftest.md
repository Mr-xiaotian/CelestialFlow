# conftest.py テスト設定説明

> 📅 最終更新日: 2026/04/22

## テスト目的

`conftest.py` は pytest のローカルプラグインファイルであり、テストセッション開始前にプロジェクトレベルの環境変数設定を読み込み、すべてのテストケースが一貫した外部サービス接続パラメータ（Redis、CelestialTree、Reporter など）を共有することを保証します。

## テスト範囲

- **環境初期化**: pytest がテストを収集する前に自動実行されます。
- **設定読み込み**: `python-dotenv` を通じてプロジェクトルートディレクトリの `.env` ファイルを読み込みます。
- **ゼロ侵入設計**: fixture 定義やフック関数を含まず、最小限の責務を維持します。

## 依存関係

| 依存パッケージ | 用途 |
|---------------|------|
| `python-dotenv` | `.env` ファイルから環境変数を読み込み |

## 起こりうる問題と注意事項

### 1. `.env` ファイルの欠落
プロジェクトルートディレクトリに `.env` ファイルがない場合、`load_dotenv()` はエラーを出さずにサイレントにスキップします。つまり、環境変数に依存するテスト（`demo_stages.py` の Redis テストなど）は、設定が空であるために失敗またはスキップされる可能性があります。

**推奨**: CI/CD 環境では `.env` ファイルに依存せず、環境変数を直接注入してください。

### 2. 環境変数の汚染
`load_dotenv()` はデフォルトで既存の環境変数を上書きしません（`override=False`）。ホストマシンに同名の変数（例: `REDIS_HOST`）が既に定義されている場合、実際に使用される値が `.env` ファイルの値と異なる可能性があります。

**トラブルシューティング**:
```bash
pytest tests/ --collect-only -v
# 環境変数の実際の値を確認
python -c "import os; print(os.getenv('REDIS_HOST'))"
```

### 3. uv 仮想環境との互換性
Windows で `uv run` を使用する際、仮想環境が破損していると `dotenv` の読み込みが失敗する可能性があります。この場合、`.venv` の整合性を確認するか、`python -m pytest` で直接実行してください。

## 実行方法

このファイルは手動で実行する必要はなく、pytest が自動的に読み込みます：
```bash
pytest tests/
```
