# Observability モジュール

Observability モジュールは、CelestialFlow のオブザーバビリティ機能を提供します。実行状態の監視、パフォーマンスメトリクスの収集、エラー追跡、およびリモート制御が含まれます。タスク実行プロセスを透明で、監視可能で、制御可能にします。

## モジュール概要

Observability モジュールは、システムの実行状態の収集、集約、および報告を担当し、リアルタイムの監視ビューとリモート制御機能を提供します。このモジュールを通じて、ユーザーはタスクの実行状態、パフォーマンスメトリクス、およびエラー情報をリアルタイムで監視し、システムの動作を動的に調整できます。

## ファイル説明

### コアコンポーネント

1. **core_report.py** (`TaskReporter`)
   - **役割**: 実行状態を収集してリモート Web サーバーに報告するタスク状態レポーター
   - **主な機能**:
     - **状態報告**: タスクグラフの構造、トポロジー、実行状態、エラー情報を定期的にプッシュします
     - **タスク注入**: Web UI からユーザーが注入した新しいタスクを受信し、実行中のタスクグラフに動的に挿入します
     - **パラメータ調整**: サーバーから設定を取得し、報告間隔などのパラメータを動的に調整します
     - **エラー同期**: 2つのエラープッシュモード（メタデータモードとコンテンツモード）をサポートします
   - **通信プロトコル**: HTTP
   - **データフォーマット**: JSON

2. **core_progress.py** (`TaskProgress`, `NullTaskProgress`)
   - **役割**: `tqdm` ベースのタスク進捗可視化
   - **主な機能**:
     - 総タスク数の動的更新（`add_total`）
     - 通常モード（`tqdm`）と非同期モード（`tqdm.asyncio`）
     - プログレスバーが無効時の `NullTaskProgress` Null 実装

## モジュール関連

### 内部関連
- `TaskReporter` は Observability モジュールの唯一のコアコンポーネントです
- プラグイン可能に設計されており、必要に応じて有効化・無効化できます

### 外部関連
- **Graph モジュール**: タスクグラフの構造とトポロジー情報を収集します
- **Runtime モジュール**: 実行状態、パフォーマンスメトリクス、およびエラー情報を収集します
- **Stage モジュール**: タスクノードの実行状態と結果を監視します
- **Persistence モジュール**: 永続化されたログおよびエラーデータを取得します
- **Web モジュール**: Web UI との双方向通信をサポートし、状態表示とリモート制御を実現します

## アーキテクチャの特徴

### 双方向通信
- **上り通信**: 状態データを Web サーバーに報告します
- **下り通信**: 制御コマンドを Web サーバーから実行中のインスタンスに送信します
- **リアルタイム**: リアルタイムの状態更新と即時制御をサポートします

### 増分更新
- 状態データを増分的に報告し、ネットワークトラフィックを削減します
- エラーログを増分的に同期し、重複送信を回避します
- 設定変更を増分的に適用し、再起動の必要性を減らします

### フォールトトレラント設計
- ネットワーク中断時のローカルキャッシュとリトライ
- データ整合性の検証と修復
- メイン実行フローに影響を与えないグレースフルデグラデーション

### セキュリティ
- 本番環境では認証レイヤーを持つリバースプロキシの使用を推奨します
- 機密データは送信されません。Web サーバーのアクセス範囲を制限することを推奨します

## データモデル

### 状態データ
- **グラフ構造**: ノードリスト、依存関係、設定パラメータ
- **実行状態**: ノード状態（待機中、実行中、完了、失敗）、進捗率
- **パフォーマンスメトリクス**: 実行時間、スループット、リソース使用率、キュー長
- **エラー情報**: エラータイプ、発生時刻、スタックトレース、影響範囲

### 制御コマンド
- **タスク注入**: 新しいタスク定義、挿入位置、依存関係
- **パラメータ調整**: 報告間隔、ログレベル、パフォーマンス閾値
- **操作コマンド**: 一時停止、再開、再起動、終了
- **設定更新**: ランタイム設定、ビジネスパラメータ、リソース制限

## 使用パターン

### 基本設定
```python
from celestialflow.observability import TaskReporter
from celestialflow.persistence import LogInlet

# レポーターを作成
reporter = TaskReporter(
    host="127.0.0.1",  # Web サーバーホスト
    port=5000,         # Web サーバーポート
    task_graph=my_task_graph,
    log_inlet=log_inlet  # LogInlet インスタンス
)

# レポーターを開始
reporter.start()

# タスク実行中、レポーターは自動的に状態を収集して報告します
```

### 高度な機能
1. **カスタムデータ収集**: カスタムデータコレクターを実装して監視ディメンションを拡張します
2. **マルチサーバー報告**: 冗長性のために複数の監視サーバーに同時に報告します
3. **ローカルストレージ**: ネットワークが利用できないときにデータをローカルに一時保存し、回復後に同期します
4. **データフィルタリング**: データフィルタリングルールを設定し、不要なデータ送信を削減します
5. **アラート統合**: 外部アラートシステムと統合し、自動アラートを実現します

### Web UI 統合
1. **リアルタイム監視**: Web UI を通じてタスク実行状態をリアルタイムで表示します
2. **履歴分析**: 過去の実行記録とパフォーマンストレンドを表示します
3. **エラー診断**: エラーの詳細と根本原因分析を表示します
4. **リモート制御**: Web UI を通じてシステムの動作を動的に調整します
5. **レポート生成**: 実行レポートとパフォーマンス分析を自動生成します

## デプロイメントの考慮事項

### ネットワーク設定
- **帯域幅要件**: データ量と報告頻度に基づいて帯域幅のニーズを評価します
- **レイテンシ許容度**: 適切なタイムアウトとリトライ戦略を設定します
- **ファイアウォール**: 監視ポートがアクセス可能であることを確認します
- **プロキシサポート**: プロキシサーバーを介した通信をサポートします

### セキュリティ設定
- **証明書管理**: TLS 証明書と鍵を設定します
- **認証メカニズム**: API キー、トークン、またはその他の認証方法を設定します
- **アクセス制御**: IP ホワイトリストとレート制限を設定します
- **データ暗号化**: 機密データの送受信および保存時の暗号化

### パフォーマンス最適化
- **バッチ処理**: データをバッチで報告し、リクエスト数を削減します
- **圧縮**: データを圧縮して送信し、帯域幅使用量を削減します
- **キャッシュ**: 頻繁にアクセスされるデータをローカルにキャッシュします
- **非同期処理**: ノンブロッキングのデータ収集と報告

## ベストプラクティス

### 本番環境
1. **高可用性デプロイメント**: 単一障害点を避けるために複数の監視サーバーをデプロイします
2. **データ保持ポリシー**: 要件に基づいてデータ保持期間を設定します
3. **キャパシティプランニング**: 監視データ量に基づいてストレージとコンピューティングリソースを計画します
4. **災害復旧**: ネットワーク中断やサーバー障害に対する対応計画を策定します

### 開発とテスト
1. **ローカルシミュレーション**: 開発環境ではローカルモックサーバーを使用します
2. **データ匿名化**: テスト環境では匿名化されたデータを使用します
3. **パフォーマンステスト**: 大規模データ報告のパフォーマンス影響をテストします
4. **障害テスト**: ネットワーク中断やサーバー障害からの回復能力をテストします

### 監視とメンテナンス
1. **自己監視**: レポーター自身の運用状態を監視します
2. **データ品質**: 報告データの完全性と正確性を監視します
3. **パフォーマンス監視**: 報告のレイテンシ、成功率、その他のメトリクスを監視します
4. **セキュリティ監査**: アクセスログとセキュリティイベントを定期的に監査します
