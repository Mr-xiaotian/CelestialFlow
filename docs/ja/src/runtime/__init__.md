# Runtime モジュール

Runtime モジュールは、CelestialFlow のタスク実行ランタイム環境を提供します。タスクスケジューリング、キュー管理、エラーハンドリング、パフォーマンス監視などのコア機能が含まれます。タスクの実際の実行を支えるインフラストラクチャ層です。

## モジュール概要

Runtime モジュールは、タスクの送信から結果の返却までのライフサイクル全体を管理します。複数の実行モード（シリアル、パラレル、非同期）、堅牢なエラーハンドリングメカニズム、パフォーマンス監視、リソース管理機能を提供します。

## ファイル説明

### コアランタイムコンポーネント

1. **core_dispatch.py** (`TaskDispatch`)
   - **役割**: 複数の実行モードをサポートするタスク実行のコアランナー
   - **実行モード**:
     - シリアル実行: タスクを順次実行します
     - スレッドプール: I/O バウンドタスクの並行実行
     - プロセスプール: CPU バウンドタスクの並行実行
     - 非同期実行: asyncio ベースの非同期タスク
   - **主要機能**: タスクスケジューリング、ワーカースレッド管理、タイムアウト制御、リソースクリーンアップ

2. **core_queue.py** (`TaskInQueue`, `TaskOutQueue`)
   - **役割**: ノード間のデータ転送を実現するタスク入出力キュー
   - **キュータイプ**:
     - `TaskInQueue`: 上流ノードの出力を受け取るタスク入力キュー
     - `TaskOutQueue`: 下流ノードに結果を送信するタスク出力キュー
   - **主要機能**: スレッドセーフティ、フロー制御、バックプレッシャー処理、シリアライズサポート

3. **core_envelope.py** (`TaskEnvelope`)
   - **役割**: タスクデータ、メタデータ、実行コンテキストを含むタスクデータラッパー
   - **含まれる情報**: タスクID、入力データ、実行パラメータ、エラーハンドリング戦略、優先度
   - **主要機能**: データカプセル化、コンテキスト伝搬、エラー伝搬、優先度ソート

### 監視とメトリクス

4. **core_metrics.py** (`TaskMetrics`)
   - **役割**: タスク実行メトリクスの収集と監視
   - **収集メトリクス**: 実行時間、成功率、エラー率、キュー長、スループット
   - **主要機能**: リアルタイム監視、パフォーマンス分析、ボトルネック検出、アラートトリガー

### ユーティリティ

5. **util_errors.py** (`CelestialFlowError`, `ConfigurationError`, `InvalidOptionError`, `ExecutionModeError`, `StageModeError`, `LogLevelError`, `RemoteWorkerError`, `UnconsumedError`, `PickleError`)
   - **役割**: 標準エラータイプと処理戦略を定義するエラーハンドリングフレームワーク
   - **エラータイプ**: 
     - `CelestialFlowError`: すべてのカスタム例外の基底クラス
     - `ConfigurationError`: 設定エラー（不正なパラメータ、サポートされていない組み合わせなど）
     - `InvalidOptionError`: 設定項目の値が不正
     - `ExecutionModeError`: 実行モードエラー
     - `StageModeError`: Stage モードエラー
     - `LogLevelError`: ログレベルエラー
     - `RemoteWorkerError`: リモートワーカーエラー
     - `UnconsumedError`: 未消費タスクエラー
     - `PickleError`: シリアライズエラー
   - **主要機能**: エラー分類、詳細なエラーメッセージ、パラメータバリデーション、エラーリカバリー

6. **util_factories.py**
   - **役割**: ランタイムコンポーネントのファクトリ関数、オブジェクト作成を簡素化
   - **ファクトリ関数**:
     - `make_counter()`: カウンターを作成（スレッドセーフ/プロセスセーフ）
     - `make_queue_backend()`: 実行モードに基づいてキューバックエンドを作成
     - `make_task_in_queue()`: タスク入力キューを作成
     - `make_task_out_queue()`: タスク出力キューを作成
   - **主要機能**: 統一されたオブジェクト作成インターフェース、設定適応、依存関係管理

7. **util_types.py**
   - **役割**: ランタイムの型定義とデータ構造
   - **含まれる型**:
     - `TerminationSignal`: タスクキュー終了のセンチネルオブジェクト
     - `TerminationIdPool`: 終了シグナル ID プール
     - `StageStatus`: Stage ステータス列挙型（NOT_STARTED, RUNNING, STOPPED）
     - `STAGE_STYLE`: Stage スタイル列挙型
     - `NodeLabelStyle`: ノードラベルスタイル（CelestialTree より）
   - **主要機能**: 型定義、列挙型管理、データ構造、型安全性

8. **util_queue.py**
   - **役割**: 汎用機能を提供するランタイムユーティリティ関数
   - **主要関数**: `cleanup_mpqueue()`: マルチプロセスキューをクリーンアップし、リソースの解放を確保します
   - **主要機能**: リソース管理、キュークリーンアップ、メモリ解放、マルチプロセスセーフティ

9. **util_hash.py**
    - **役割**: タスクの重複排除とキャッシュのためのオブジェクトハッシュ計算
    - **主要関数**:
      - `make_hashable()`: オブジェクトをハッシュ可能な形式に変換します
      - `object_to_str_hash()`: オブジェクトの文字列ハッシュ値を計算します
    - **主要機能**: 安定したハッシュ計算、オブジェクトシリアライズ、タスク重複排除、キャッシュキー生成

10. **util_estimators.py**
    - **役割**: 実行時間の推定と進捗計算
    - **主要関数**:
      - `calc_elapsed()`: 経過時間を計算します
      - `calc_remaining()`: 残り時間を計算します
      - `calc_global_remain_equal_pred()`: グローバル残り時間を計算します（等重み予測）
    - **主要機能**: 時間推定、進捗予測、パフォーマンス分析、ETA 計算

## モジュール関連

### 内部関連
- `TaskDispatch` は `TaskInQueue` と `TaskOutQueue` を使用してタスクの取得と結果の送信を行います
- `TaskEnvelope` はキューを通じて渡され、完全な実行コンテキストを保持します
- `TaskMetrics` は `TaskDispatch` の実行状態を監視します
- すべてのエラーは `CelestialFlowError` とそのサブクラスを通じて統一的に処理されます

### 外部関連
- **Stage モジュールとの関連**: `TaskDispatch` は `TaskExecutor` と `TaskStage` を実行します
- **Graph モジュールとの関連**: `TaskGraph` に実行エンジンと通信メカニズムを提供します
- **Persistence モジュールとの関連**: 実行状態の永続化をサポートします
- **Observability モジュールとの関連**: 監視データとパフォーマンスメトリクスを提供します

## アーキテクチャの特徴

### マルチモード実行
- さまざまなシナリオの要件に対応する4つの実行モードをサポートします
- 最適な実行戦略を自動的に選択します
- ハイブリッド実行モードをサポートします

### 堅牢性設計
- 完全なエラーハンドリングチェーン
- 自動リトライとフェイルオーバー
- リソースリーク防止

### 可観測性
- 包括的なメトリクス収集
- リアルタイムの進捗レポート
- 詳細な実行ログ

### 拡張性
- プラグインベースのアーキテクチャ
- カスタム実行戦略
- 設定可能なキュー実装

## 使用パターン

### 基本的な使用方法
1. **ランナーの作成**: タスクタイプに基づいて適切な `TaskDispatch` モードを選択します
2. **キューの設定**: 入出力キューを設定し、データフローチャネルを確立します
3. **タスクの実行**: ランナーにタスクを送信し、実行状態を監視します
4. **結果の処理**: 出力キューから結果を取得し、エラーを処理します

### 高度な使用方法
1. **カスタムランナー**: `TaskDispatch` を継承して特定の実行ロジックを実装します
2. **ハイブリッドモード**: 複雑なワークフローに対して異なる実行モードを組み合わせます
3. **監視統合**: 外部監視システムと統合し、集中監視を実現します

## ベストプラクティス

1. **I/O バウンドタスク**: スレッドプールモードを使用します
2. **CPU バウンドタスク**: プロセスプールモードを使用します
3. **非同期タスク**: 非同期モードを使用して並行性能を向上させます
4. **クリティカルタスク**: 適切なリトライ戦略とタイムアウト設定を構成します
5. **バッチタスク**: バッチ処理モードを使用してスループットを向上させます
6. **監視設定**: 本番環境向けに包括的な監視とアラートを構成します
