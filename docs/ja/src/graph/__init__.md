# Graph モジュール

Graph モジュールは CelestialFlow のコアスケジューリングシステムで、タスクノード間の依存関係、実行フロー、ライフサイクルの管理を担当します。柔軟なタスクグラフの構築、分析、シリアライズ機能を提供します。

## モジュール概要

Graph モジュールはタスク実行の基本単位とそれらの関係を定義し、有向非巡回グラフ（DAG）を形成します。各ノードは `TaskStage` を表し、エッジは依存関係を表します。このモジュールはタスクが正しい順序で実行されることを保証し、並行処理、エラー処理、リソース管理を処理します。

## ファイル説明

### コアファイル

1. **core_graph.py** (`TaskGraph`)
   - **役割**: `TaskStage` ノードの依存関係、実行フロー、リソース割り当て、ライフサイクルを管理するコアスケジューラーです
   - **主な機能**:
     - ノード間の依存関係の確立
     - タスクグラフの実行（同期/非同期）
     - エラー処理とリトライメカニズム
     - リソース管理と並行制御

2. **core_structure.py** (`TaskChain`, `TaskLoop`, `TaskCross`, `TaskComplete`, `TaskWheel`, `TaskGrid`)
   - **役割**: 一般的なパターンの構築を簡素化するための事前定義されたタスクグラフ構造を提供します
   - **含まれる構造**:
     - `TaskChain`: ノードが順番に実行される線形タスクチェーン
     - `TaskLoop`: 条件付きループをサポートするループ構造
     - `TaskCross`: レイヤー間でノードが相互接続されるクロス接続構造
     - `TaskComplete`: すべてのノードが相互接続される完全接続構造
     - `TaskWheel`: 中心ノードが他のすべてのノードに接続される車輪構造
     - `TaskGrid`: ノードが行と列に配置されるグリッド構造

### ユーティリティファイル

3. **util_analysis.py**
   - **役割**: グラフ理論に基づく分析と診断機能を提供するタスクグラフ分析ツールです
   - **主な関数**:
     - `format_networkx_graph()`: 構造グラフを networkx 有向グラフに変換します
     - `compute_node_levels()`: グラフ内のノードレベルを計算します（トポロジカルソート）
   - **主な機能**:
     - グラフ構造の変換と可視化サポート
     - ノードレベルの計算とトポロジカル分析
     - 依存関係の分析と検証

4. **util_serialize.py**
   - **役割**: タスクグラフのシリアライズと構造構築ツールです
   - **主な関数**:
     - `build_structure_graph()`: ルートノードから JSON グラフ構造を構築します
     - `_build_structure_subgraph()`: 単一のサブグラフ構造を構築します（内部関数）
     - `format_structure_list_from_graph()`: グラフ構造から文字列リストをフォーマットします
   - **主な機能**:
     - タスクグラフ構造の JSON 形式へのシリアライズ
     - 再帰的なグラフ構造の構築
     - 構造のフォーマットとテキスト出力

## モジュールの関連

### 内部の関連
- `TaskGraph` は基底クラスであり、他のすべての構造はそれを継承または使用します
- `TaskChain`、`TaskLoop` などは `TaskGraph` の特化実装です
- 分析ツールは診断のためにグラフ構造に依存します
- シリアライズツールは任意の `TaskGraph` インスタンスを永続化できます

### 外部との関連
- **Stage モジュールとの関連**: `TaskStage` ノードを管理し、各ノードは実行可能な単位です
- **Runtime モジュールとの関連**: タスク実行に `TaskDispatch` を使用し、通信に `TaskQueue` に依存します
- **Persistence モジュールとの関連**: シリアライズツールを通じて永続化ストレージと連携します
- **Observability モジュールとの関連**: 実行状態とパフォーマンスメトリクスを提供します

## 使用パターン

1. **タスクグラフの構築**: `TaskStage` ノードを作成し、`graph.connect()` で依存関係を確立します
2. **構造の選択**: 要件に基づいて事前定義された構造を使用します（例：線形フローには `TaskChain`）
3. **実行**: `start_graph()` を呼び出すか、`start_chain()` などのサブクラスメソッドでタスクグラフを実行します
4. **監視**: 分析ツールを使用してグラフ構造を検査し、実行状態を監視します
5. **永続化**: 必要に応じてシリアライズツールを使用してタスクグラフを保存/読み込みします

## ベストプラクティス

- シンプルな線形フローには `TaskChain` を優先して使用してください
- 複雑な分岐ロジックには `TaskSplitter` と `TaskRouter` を使用してください
- 循環タスクには `TaskLoop` を使用してください
- 定期的に分析ツールを使用してグラフ構造の健全性を確認してください
- デバッグとリカバリのために重要なワークフローをシリアライズして保存することをお勧めします
