# Graph モジュール

> 📅 最終更新日: 2026/04/22

Graph モジュールは CelestialFlow のコアスケジューリングシステムであり、タスクノード間の依存関係、実行フロー、ライフサイクルの管理を担当します。柔軟なタスクグラフの構築、分析、シリアライズ機能を提供します。

## モジュール概要

Graph モジュールはタスク実行の基本単位とその関係を定義し、有向非巡回グラフ（DAG）を形成します。各ノードは `TaskStage` を表し、エッジは依存関係を表します。このモジュールはタスクが正しい順序で実行されることを保証し、並行処理、エラーハンドリング、リソース管理を処理します。

## ファイル説明

### コアファイル

1. **core_graph.py** (`TaskGraph`)
   - **役割**: コアスケジューラ。`TaskStage` ノードの依存関係、実行フロー、リソース割り当て、ライフサイクルを管理
   - **主要機能**:
     - ノード間の依存関係の構築
     - タスクグラフの実行（同期/非同期）
     - エラーハンドリングとリトライメカニズム
     - リソース管理と並行制御

2. **core_structure.py** (`TaskChain`, `TaskLoop`, `TaskCross`, `TaskComplete`, `TaskWheel`, `TaskGrid`)
   - **役割**: 定義済みのタスクグラフ構造を提供し、一般的なパターンの構築を簡素化
   - **含まれる構造**:
     - `TaskChain`: 線形タスクチェーン、ノードが順次実行
     - `TaskLoop`: ループ構造、条件付きループをサポート
     - `TaskCross`: クロス接続構造、レイヤー間でノードが相互接続
     - `TaskComplete`: 完全接続構造、すべてのノードが相互接続
     - `TaskWheel`: ホイール構造、中心ノードが他のすべてのノードに接続
     - `TaskGrid`: グリッド構造、ノードが行列に配置

### ユーティリティファイル

3. **util_analysis.py**
   - **役割**: グラフ理論分析と診断を提供するタスクグラフ分析ツール
   - **主要関数**:
     - `format_networkx_graph()`: 構造グラフを networkx 有向グラフに変換
     - `compute_node_levels()`: グラフ内のノードレベルを計算（トポロジカルソート）
   - **主要機能**:
     - グラフ構造の変換と可視化サポート
     - ノードレベルの計算とトポロジー分析
     - 依存関係の分析と検証

4. **util_serialize.py**
   - **役割**: タスクグラフのシリアライズと構造構築ツール
   - **主要関数**:
     - `build_structure_graph()`: ルートノードから JSON グラフ構造を構築
     - `_build_structure_subgraph()`: 単一のサブグラフ構造を構築（内部関数）
     - `format_structure_list_from_graph()`: グラフ構造から文字列リストをフォーマット
   - **主要機能**:
     - タスクグラフ構造の JSON 形式へのシリアライズ
     - 再帰的なグラフ構造表現の構築
     - 構造のフォーマットとテキスト出力

## モジュール関連

### 内部関連
- `TaskGraph` は基底クラスであり、他のすべての構造はこれを継承または使用
- `TaskChain`、`TaskLoop` などは `TaskGraph` の特化実装
- 分析ツールは診断のためにグラフ構造に依存
- シリアライズツールは任意の `TaskGraph` インスタンスを永続化可能

### 外部関連
- **Stage モジュール**: `TaskStage` ノードを管理。各ノードは実行可能な単位
- **Runtime モジュール**: `TaskDispatch` を使用してタスクを実行し、`TaskQueue` を通じて通信
- **Persistence モジュール**: シリアライズツールを通じて永続ストレージと連携
- **Observability モジュール**: 実行状態とパフォーマンスメトリクスを提供

## 使用パターン

1. **タスクグラフの構築**: `TaskStage` ノードを作成し、`graph.connect()` で依存関係を構築
2. **構造の選択**: 要件に基づいて定義済み構造を使用（例：線形フローには `TaskChain`）
3. **実行**: `start_graph()` またはサブクラスの `start_chain()` などのメソッドでタスクグラフを実行
4. **監視**: 分析ツールを使用してグラフ構造を検査し、実行状態を監視
5. **永続化**: 必要に応じてシリアライズツールでタスクグラフを保存/読み込み

## ベストプラクティス

- シンプルな線形フローには `TaskChain` を優先使用
- 複雑な分岐ロジックには `TaskSplitter` と `TaskRouter` を使用
- 循環タスクには `TaskLoop` を使用
- 定期的に分析ツールを使用してグラフ構造の健全性を確認
- 重要なワークフローはシリアライズして保存し、デバッグとリカバリに活用
