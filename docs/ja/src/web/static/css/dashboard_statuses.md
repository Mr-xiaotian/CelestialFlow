# dashboard_statuses.css

> 📅 最終更新日: 2026/06/22

ダッシュボードノード状態カードのレイアウトとスタイル定義を担当し、統計グリッド、4 セグメントプログレスバー、およびノード状態に基づく動的ボーダー色を含みます。

## レイアウト構造

### 統計グリッド (`.stat-grid`)
- `grid` レイアウトを使用し、固定で 2 列に設定。
- 成功、待機、エラー、重複などのコア指標を表示します。

### ノードカード (`.node-card`)
- 角丸カードデザインを採用（`border-radius: 1rem`）。
- **状態ボーダー**: 左側に 3px 幅の状態バーを設置し、色はノードの実行状態に応じて動的に変化します：
  - `.status-running`: `--cornflower-400`（青）を使用、実行中を示します。
  - `.status-stopped`: `--carbon-400`（灰色）を使用、停止済みを示します。
  - デフォルト: `--carbon-300` を使用、未起動を示します。

## プログレスバーレンダリング (`.progress-bar`)

プログレスバーは 4 つのセグメント（`.progress-segment`）で構成され、それぞれ異なるタスク状態色に対応します：

| クラス名 | 対応指標 | ライトモード色 | ダークモード色 |
|------|----------|--------------|--------------|
| `.seg-success` | 成功 | `--jade-400` | `--jade-700` |
| `.seg-error` | エラー | `--crimson-400` | `--crimson-700` |
| `.seg-duplicate` | 重複 | `--marigold-400` | `--marigold-700` |
| `.seg-pending` | 待機 | `--carbon-300` | `--carbon-600` |

## 時間推定エリア (`.time-estimate`)

- 等幅フォント（`monospace`）を使用して整列を確保します。
- `.elapsed` シリーズクラス名は、実行済み時間の各桁をセグメント別に着色するために使用されます（成功、エラー、重複）。

## インタラクション効果

- **クリックフィードバック**:
  - `.error-clickable`: エラー数値項目にポインターカーソル（`pointer`）を表示し、クリックでジャンプ可能であることを示します。

## レスポンシブデザイン

- 幅が `2048px` 未満の場合、`#dashboard-grid` は単一カラムレイアウトに切り替わります。
- 長いタイトルの折り返しロジックを自動処理します。
