# Web モジュール

> 📅 最終更新日: 2026/04/22

Web モジュールは、CelestialFlow の Web ベースの監視・管理インターフェースを提供します。FastAPI をベースに構築されており、リアルタイムのタスクステータス表示、エラー監視、タスクインジェクション、リモート制御機能をサポートします。

## モジュール概要

Web モジュールは独立した Web サーバーであり、CelestialFlow の視覚的な監視・管理インターフェースを提供します。`TaskReporter` からのステータスデータを受信し、リアルタイム監視ビューを提供し、Web インターフェースを通じたタスクインジェクションおよびシステム制御をサポートします。

## ファイル説明

### コアコンポーネント

1. **core_server.py** (`TaskWebServer`)
   - **役割**: RESTful API エンドポイントを提供する FastAPI ベースの Web サーバー
   - **主要機能**:
     - **ステータス表示**: タスクグラフ構造、ステージ状態、パフォーマンス指標のリアルタイム表示
     - **エラー監視**: エラー詳細、スタックトレース、発生タイムラインの表示
     - **タスクインジェクション**: Web インターフェースから実行中のタスクグラフに新しいタスクを動的にインジェクション
     - **リモート制御**: 一時停止、再開、再起動、終了などの制御操作をサポート
     - **履歴分析**: 過去の実行記録とパフォーマンストレンドの閲覧
   - **技術スタック**: FastAPI、Jinja2 テンプレート、静的ファイルサービング

### ユーティリティコンポーネント

2. **util_cal.py**
   - **役割**: ミリ秒のリフレッシュ間隔を秒に変換し範囲を制限する計算ユーティリティ関数

3. **util_config.py**
   - **役割**: フロントエンド設定ファイル（JSON）の読み込みと保存

4. **util_error.py**
   - **役割**: エラークエリパラメータの正規化、フィルタリング、ページネーションのユーティリティ関数

## モジュール連携

### 内部連携
- `TaskWeb` は Web モジュールの唯一のコアコンポーネントです
- フロントエンド静的ファイル（HTML、CSS、JavaScript）とバックエンド API サービスを含みます

### 外部連携
- **Observability モジュールとの連携**: `TaskReporter` から報告されたステータスデータを受信
- **Graph モジュールとの連携**: タスクグラフ構造とステージ状態を表示
- **Runtime モジュールとの連携**: 実行状態とパフォーマンス指標を表示
- **Stage モジュールとの連携**: タスクステージの詳細と実行結果を表示
- **Persistence モジュールとの連携**: 永続化されたログとエラーデータにアクセス

## アーキテクチャ特徴

### フロントエンド・バックエンド分離
- **バックエンド**: FastAPI が RESTful API サービスを提供
- **フロントエンド**: モダンブラウザをサポートする静的 HTML/JS/CSS ファイル
- **通信**: JSON over HTTP、定期ポーリングによるステータス更新

### リアルタイム対応
- **定期ポーリング**: フロントエンドが設定可能な間隔でステータス更新を定期的にリクエスト
- **バージョンベースキャッシュ**: `known_rev` パラメータをサポートし、変更がない場合は `data=null` を返却して転送量を削減

### 拡張性
- **プラグインアーキテクチャ**: カスタム監視パネルとコントロールをサポート
- **テーマシステム**: インターフェースのテーマ切り替えとカスタマイズをサポート
- **多言語対応**: 国際化インターフェースをサポート
- **権限システム**: ロールベースのアクセス制御

### セキュリティ
- **CORS 設定**: クロスオリジンアクセスを制御
- **認証・認可**: 複数の認証方式をサポート（API Key、OAuth、JWT）
- **入力バリデーション**: 厳格な入力データの検証とサニタイズ
- **レート制限**: API の不正使用を防止

## 機能特性

### 監視ダッシュボード
- **トポロジビュー**: タスクグラフの構造と依存関係を視覚的に表示
- **ステータスパネル**: ステージ状態のリアルタイム表示（色分け）
- **パフォーマンスダッシュボード**: CPU、メモリ、キュー長などの指標を表示
- **エラーセンター**: エラー情報と統計の一元表示

### 制御機能
- **タスクインジェクション**: フォームまたは JSON で新しいタスクを定義
- **パラメータ調整**: システムパラメータと設定を動的に調整
- **実行制御**: タスク実行の開始、一時停止、再開、終了
- **データエクスポート**: 実行ログ、パフォーマンスデータ、エラーレポートのエクスポート

### 分析機能
- **履歴トレンド**: パフォーマンス指標の履歴変化トレンドを閲覧
- **比較分析**: 異なる期間の実行状況を比較
- **根本原因分析**: エラーの根本原因分析と相関分析
- **レポート生成**: 実行レポートとパフォーマンス分析レポートの自動生成

## 使用パターン

### サーバーの起動
```bash
# コマンドラインで起動
celestialflow-web

# ポートとホストを指定
celestialflow-web --host 0.0.0.0 --port 5080
```

### API の使用
```python
import requests

# タスクグラフのステータスを取得
response = requests.get("http://localhost:5000/api/pull_status")
status = response.json()

# 新しいタスクをインジェクション（push_injection_tasks に POST し、TaskGraph が取得）
task_injection = {
    "node": "Stage 1",
    "task_datas": [[1, 2, 3]],
    "timestamp": "2024-01-01T00:00:00"
}
response = requests.post("http://localhost:5000/api/push_injection_tasks", json=task_injection)
```

## デプロイ考慮事項

### サーバー設定
- **プロセス管理**: Gunicorn、Uvicorn などの WSGI/ASGI サーバーを使用
- **ロードバランシング**: マルチインスタンスデプロイ時にロードバランシングを設定
- **リバースプロキシ**: Nginx、Apache をリバースプロキシとして使用
- **SSL/TLS**: HTTPS を設定してデータ転送のセキュリティを確保

### パフォーマンス最適化
- **静的ファイルキャッシュ**: ブラウザキャッシュ戦略を設定
- **API キャッシュ**: アクセス頻度の高い API レスポンスをキャッシュ
- **コネクションプール**: データベースおよび外部サービスのコネクションプール
- **圧縮転送**: Gzip/Brotli 圧縮を有効化

### 高可用性
- **マルチインスタンスデプロイ**: 複数の Web サーバーインスタンスをデプロイ
- **セッション共有**: 共有セッションストレージ（Redis）を設定
- **ヘルスチェック**: ヘルスチェックエンドポイントを設定
- **フェイルオーバー**: 自動フェイルオーバーメカニズムを設定
