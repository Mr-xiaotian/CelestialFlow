# Persistence モジュール

Persistence モジュールは、CelestialFlow のデータ永続化機能を提供します。実行ログの記録、エラー情報の保存、タスク状態の保存、および設定定数の管理が含まれます。タスク実行の重要なデータが確実に保存・取得できることを保証します。

## モジュール概要

Persistence モジュールは、ランタイムデータをさまざまなストレージバックエンドに永続化する役割を担い、ログ記録、エラー追跡、状態スナップショット、および設定管理をサポートします。モジュールはスパウトパターンを採用しており、メインフローのパフォーマンスに影響を与えることなく、タスク実行フローにシームレスに統合されます。

## ファイル説明

### ログ永続化

1. **core_log.py** (`LogSpout`, `LogInlet`)
   - **役割**: ログの記録と保存のためのインフラストラクチャ
   - **コアコンポーネント**:
     - `LogSpout`: キューからログメッセージを受信してファイルに書き込むログスパウト
     - `LogInlet`: ログをファイルに書き込むログコレクター
   - **ログフォーマット**: タイムスタンプ、ログレベル、メッセージ内容を含むテキストログフォーマット
   - **主な機能**: 非同期書き込み、ファイル管理、ログローテーション、マルチプロセス安全性

### エラー永続化

2. **core_fail.py** (`FailSpout`, `FailInlet`)
   - **役割**: エラー情報の記録と保存のためのインフラストラクチャ
   - **コアコンポーネント**:
     - `FailSpout`: キューからエラー情報を受信してファイルに書き込むエラースパウト
     - `FailInlet`: エラー情報をファイルに書き込むエラーコレクター
   - **エラー情報**: エラータイプ、スタックトレース、コンテキストデータ、発生時刻、タスク ID
   - **主な機能**: エラー分類、ファイルストレージ、非同期処理、マルチプロセス安全性

### 成功結果の永続化

3. **core_success.py** (`SuccessSpout`)
   - **役割**: 成功結果キューを継続的に読み取り、task-result ペアをキャッシュする成功結果リスナースレッド
   - **コアコンポーネント**:
     - `SuccessSpout`: `BaseSpout` を継承し、(task, result) ペアをキャッシュします
   - **主な機能**: 成功結果のキャッシュ、task-result ペアの抽出

### データフォーマットと設定

4. **util_jsonl.py**
   - **役割**: 効率的な構造化データの保存と読み取りのための JSON Lines フォーマットサポート
   - **主な機能**:
     - `load_jsonl_logs()`: JSONL ファイルからログデータを読み込み、選択的なフィールド読み取りをサポートします
     - 指定した行番号からの開始をサポートするストリーミングデータ読み取り
     - データフィルタリングと変換
     - エラーハンドリングとファイル検証
   - **使用シーン**: ログファイルの読み取り、エラーレコードの分析、Web UI データ表示

5. **util_constant.py**
   - **役割**: 永続化関連の定数と設定定義
   - **内容**:
     - `LEVEL_DICT`: ログレベル辞書。ログレベルとそれに対応する数値を定義します
     - ログレベル: TRACE(0), DEBUG(10), SUCCESS(20), INFO(30), WARNING(40), ERROR(50), CRITICAL(60)
   - **主な機能**: 統一されたログレベル管理、レベル比較、ログフィルタリング

## モジュール関連

### 内部関連
- すべての永続化クラスは `BaseSpout`/`BaseInlet`（Funnel モジュールで定義）を継承しています
- `LogSpout`/`LogInlet` と `FailSpout`/`FailInlet` はペアで使用されます
- `SuccessSpout` は独立して使用され、成功結果をキャッシュします
- ユーティリティクラスはコアクラスによって使用され、フォーマットサポートと設定管理を提供します

### 外部関連
- **Runtime モジュール**: ランタイムで生成されたログとエラーをリッスンします
- **Stage モジュール**: タスクの実行状態と結果を記録します
- **Graph モジュール**: グラフ構造の状態と実行履歴を保存します
- **Observability モジュール**: 監視と分析のための生データを提供します
- **Utils モジュール**: データ処理とフォーマッティングのためにユーティリティ関数を使用します

## アーキテクチャの特徴

### 非同期ノンブロッキング設計
- スパウトはバックグラウンドスレッドで実行され、メインフローをブロックしません
- 書き込みピークに対応するキューバッファリング
- ストレージ効率を向上させるバッチコミット

### マルチバックエンドサポート
- ファイル、データベース、リモートサービスなどの複数のストレージタイプをサポートします
- 容易な切り替えと拡張のための統一インターフェース
- カスタムバックエンドをサポートするストレージアダプターパターン

### 信頼性保証
- 書き込み失敗のリトライ機構
- データ整合性の検証
- ストレージスペースの監視とアラート
- メインフローに影響を与えないグレースフルデグラデーション

### 設定可能性
- ストレージパスは設定可能
- フォーマットとエンコーディングは調整可能
- パフォーマンスパラメータはチューニング可能
- アラートルールはカスタマイズ可能

## 使用パターン

### 基本設定
```python
from celestialflow.persistence import LogSpout, LogInlet, FailSpout, FailInlet

# ログ永続化を設定
log_spout = LogSpout()
log_spout.start()
log_inlet = LogInlet(log_spout.get_queue())

# エラー永続化を設定
fail_spout = FailSpout(error_source="graph_errors")
fail_spout.start()
fail_inlet = FailInlet(fail_spout.get_queue())
```

### 高度な使用
1. **マルチバックエンドストレージ**: 冗長ストレージのためにファイルとデータベースに同時に書き込みます
2. **カスタムフォーマット**: カスタムシリアライゼーションフォーマットを実装します
3. **リアルタイムアラート**: エラー閾値を設定してリアルタイム通知をトリガーします
4. **データアーカイブ**: 履歴データを自動的にアーカイブしてストレージスペースを解放します
5. **データマイグレーション**: 異なるストレージバックエンド間でデータを移行します

### 監視とメンテナンス
1. **ストレージ監視**: ディスク使用率を監視してストレージオーバーフローを防止します
2. **パフォーマンス監視**: 書き込みレイテンシとスループットを監視します
3. **エラー監視**: 永続化失敗率を監視します
4. **データクリーンアップ**: 期限切れデータを定期的にクリーンアップします

## ベストプラクティス

### 本番環境設定
1. **ストレージ計画**: データ量と保持ポリシーに基づいてストレージスペースを計画します
2. **冗長設計**: 重要なデータにはマルチレプリカストレージを使用します
3. **パフォーマンス最適化**: 負荷に基づいてバッチ処理サイズと並行度を調整します
4. **監視とアラート**: 包括的な監視とアラートルールを設定します

### 開発とテスト
1. **ローカルストレージ**: 開発環境ではデバッグしやすいファイルストレージを使用します
2. **モックバックエンド**: テスト環境ではインメモリまたはモックバックエンドを使用します
3. **データ分離**: 異なる環境で異なるストレージパスを使用します
4. **クリーンアップ戦略**: テストデータを自動的にクリーンアップして蓄積を防止します

### 障害処理
1. **書き込み失敗**: 適切なリトライ戦略とフォールバックプランを設定します
2. **ストレージフル**: ストレージスペースを監視し、自動クリーンアップを設定します
3. **ネットワーク障害**: リモートストレージのネットワーク中断を処理します
4. **フォーマットエラー**: データ検証とフォーマット互換性の処理
