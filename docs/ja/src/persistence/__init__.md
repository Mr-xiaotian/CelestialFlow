# Persistence モジュール

> 📅 最終更新日: 2026/04/22

Persistence モジュールは CelestialFlow のデータ永続化機能を提供し、実行ログ記録、エラー情報ストレージ、タスク状態保存、設定定数管理を含みます。タスク実行の重要なデータが確実に保存・取得できることを保証します。

## モジュール概要

Persistence モジュールはランタイムデータをさまざまなストレージバックエンドに永続化し、ログ記録、エラー追跡、状態スナップショット、設定管理をサポートします。本モジュールは Spout パターンを採用し、メインフローのパフォーマンスに影響を与えることなく、タスク実行フローにシームレスに統合されます。

## ファイル説明

### ログ永続化

1. **core_log.py** (`LogSpout`, `LogInlet`)
   - **役割**: ログ記録とストレージの基盤
   - **コアコンポーネント**:
     - `LogSpout`: ログ Spout。キューからログメッセージを受信しファイルに書き込む
     - `LogInlet`: ログコレクター。ログをファイルに書き込む
   - **ログ形式**: タイムスタンプ、ログレベル、メッセージ内容を含むテキストログ形式
   - **主要機能**: 非同期書き込み、ファイル管理、ログローテーション、マルチプロセス安全

### エラー永続化

2. **core_fail.py** (`FailSpout`, `FailInlet`)
   - **役割**: エラー情報の記録とストレージの基盤
   - **コアコンポーネント**:
     - `FailSpout`: エラー Spout。キューからエラー情報を受信しファイルに書き込む
     - `FailInlet`: エラーコレクター。エラー情報をファイルに書き込む
   - **エラー情報**: エラータイプ、スタックトレース、コンテキストデータ、発生時刻、タスク ID
   - **主要機能**: エラー分類、ファイルストレージ、非同期処理、マルチプロセス安全

### 成功結果の永続化

3. **core_success.py** (`SuccessSpout`)
   - **役割**: 成功結果の監視スレッド。成功結果キューを継続的に読み取り、task-result ペアをキャッシュ
   - **コアコンポーネント**:
     - `SuccessSpout`: `BaseSpout` を継承し、(task, result) ペアをキャッシュ
   - **主要機能**: 成功結果のキャッシュ、task-result ペアの抽出

### データ形式と設定

4. **util_jsonl.py**
   - **役割**: 効率的な構造化データの保存と読み取りのための JSON Lines 形式サポート
   - **主要機能**:
     - `load_jsonl_logs()`: JSONL ファイルからログデータを読み込み、選択的なフィールド読み取りをサポート
     - ストリーミングデータ読み取り、指定行番号からの開始をサポート
     - データフィルタリングと変換
     - エラーハンドリングとファイル検証
   - **使用シーン**: ログファイルの読み取り、エラーレコード分析、Web インターフェースのデータ表示

5. **util_constant.py**
   - **役割**: 永続化関連の定数と設定定義
   - **内容**:
     - `LEVEL_DICT`: ログレベルとそれに対応する数値を定義するログレベル辞書
     - ログレベル: TRACE(0), DEBUG(10), SUCCESS(20), INFO(30), WARNING(40), ERROR(50), CRITICAL(60)
   - **主要機能**: 統一されたログレベル管理、レベル比較、ログフィルタリング

## モジュール関連

### 内部関連
- すべての永続化クラスは `BaseSpout`/`BaseInlet`（Funnel モジュールで定義）を継承
- `LogSpout`/`LogInlet` と `FailSpout`/`FailInlet` はペアで使用
- `SuccessSpout` は独立して使用し、成功結果をキャッシュ
- ユーティリティクラスはコアクラスによって使用され、形式サポートと設定管理を提供

### 外部関連
- **Runtime モジュール**: ランタイムで生成されるログとエラーを監視
- **Stage モジュール**: タスク実行状態と結果を記録
- **Graph モジュール**: グラフ構造の状態と実行履歴を保存
- **Observability モジュール**: 監視と分析のための生データを提供
- **Utils モジュール**: データ処理とフォーマットのためのユーティリティ関数を使用

## アーキテクチャ特性

### 非同期ノンブロッキング設計
- Spout はバックグラウンドスレッドで実行し、メインフローをブロックしない
- キューバッファリングで書き込みピークに対応
- バッチコミットでストレージ効率を向上

### マルチバックエンドサポート
- ファイル、データベース、リモートサービスなど複数のストレージバックエンドをサポート
- 切り替えと拡張が容易な統一インターフェース
- カスタムバックエンドをサポートするストレージアダプターパターン

### 信頼性保証
- 書き込み失敗時のリトライメカニズム
- データ整合性の検証
- ストレージ容量の監視とアラート
- メインフローに影響を与えないグレースフルデグラデーション

### 設定可能性
- 設定可能なストレージパス
- 調整可能なフォーマットとエンコーディング
- チューニング可能なパフォーマンスパラメータ
- カスタマイズ可能なアラートルール

## 使用パターン

### 基本設定
```python
from celestialflow.persistence import LogSpout, LogInlet, FailSpout, FailInlet

# ログ永続化の設定
log_spout = LogSpout()
log_spout.start()
log_inlet = LogInlet(log_spout.get_queue())

# エラー永続化の設定
fail_spout = FailSpout(error_source="graph_errors")
fail_spout.start()
fail_inlet = FailInlet(fail_spout.get_queue())
```

### 高度な使用方法
1. **マルチバックエンドストレージ**: ファイルとデータベースに同時書き込みで冗長ストレージを実現
2. **カスタムフォーマット**: カスタムシリアライズ形式の実装
3. **リアルタイムアラート**: エラー閾値を設定してリアルタイム通知をトリガー
4. **データアーカイブ**: 履歴データを自動アーカイブしてストレージ容量を解放
5. **データ移行**: 異なるストレージバックエンド間でのデータ移行

### 監視とメンテナンス
1. **ストレージ監視**: ディスク使用率を監視してストレージオーバーフローを防止
2. **パフォーマンス監視**: 書き込みレイテンシとスループットを監視
3. **エラー監視**: 永続化失敗率を監視
4. **データクリーンアップ**: 期限切れデータを定期的にクリーンアップ

## ベストプラクティス

### 本番環境設定
1. **ストレージ計画**: データ量と保持ポリシーに基づいてストレージ容量を計画
2. **冗長設計**: 重要なデータにはマルチコピーストレージを使用
3. **パフォーマンス最適化**: 負荷に基づいてバッチ処理サイズと並行度を調整
4. **監視とアラート**: 包括的な監視とアラートルールを設定

### 開発とテスト
1. **ローカルストレージ**: 開発環境ではファイルストレージを使用してデバッグを容易に
2. **モックバックエンド**: テスト環境ではインメモリまたはモックバックエンドを使用
3. **データ分離**: 異なる環境で異なるストレージパスを使用
4. **クリーンアップ戦略**: テストデータを自動クリーンアップして蓄積を防止

### トラブルシューティング
1. **書き込み失敗**: 適切なリトライ戦略とデグラデーションプランを設定
2. **ストレージフル**: ストレージ容量を監視し自動クリーンアップを設定
3. **ネットワーク障害**: リモートストレージのネットワーク中断を処理
4. **フォーマットエラー**: データ検証とフォーマット互換性の処理
