# ログ永続化 (Log Persistence)

`celestialflow.persistence` モジュールは、マルチプロセス環境でのログの統一収集、フォーマット化、および永続化の問題を解決するために設計されたマルチプロセス安全なロギングシステムを提供します。

コアコンポーネントは `LogSpout` と `LogInlet` です。

## アーキテクチャ設計

エラー永続化と同様に、ロギングシステムも **Logger-Listener** パターンを採用しています：

1.  **LogInlet (プロデューサー)**:
    -   各 Worker プロセスが保持するラッパークラスです。
    -   豊富なセマンティックメソッド（`task_success`、`start_stage` など）を提供します。
    -   ログメッセージとレベルをパッケージ化し、マルチプロセスキュー（`multiprocessing.Queue`）に配置します。
    -   ログレベルベースのフィルタリングをサポートし、不要なクロスプロセス通信を削減します。

2.  **LogSpout (コンシューマー)**:
    -   メインプロセスの独立したデーモンスレッドで実行されます。
    -   キューからログレコードを取得し、ファイルに書き込みます。
    -   ログファイルのローテーションとフォーマットを一元管理します。

## ログレベル

システムは以下の標準ログレベルをサポートしています（値が大きいほど優先度が高い）：

-   **TRACE** (0): 最も詳細なトレース情報。キューの `put`/`get` 操作など。
-   **DEBUG** (10): デバッグ情報。プロセス終了、タスク入力など。
-   **SUCCESS** (20): 重要な操作の成功。タスク完了、分割成功など。
-   **INFO** (30): 一般情報。ステージの開始/終了、グラフ構造の出力など。
-   **WARNING** (40): 警告情報。タスクのリトライ、キュー操作の異常など。
-   **ERROR** (50): エラー情報。タスクの失敗、ループ例外など。
-   **CRITICAL** (60): 重大なエラー。

## LogSpout

`LogSpout` はログファイルの設定と書き込みスレッドを管理します。

### 初期化

```python
listener = LogSpout()
listener.start()
```

起動すると、ログは `logs/task_logger({date}).log` ファイルに書き込まれます。

### ファイルパス

```text
logs/
└── task_logger(2023-10-27).log
```

## LogInlet

`LogInlet` は、異なるコンポーネント向けの専用ログメソッドを提供し、ログ内容の構造化と一貫性を確保します。

### 初期化

```python
sinker = LogInlet(log_queue, log_level="SUCCESS")
```

-   `log_queue`: `LogSpout.get_queue()` が返すキューです。
-   `log_level`: この Inlet の最低ログレベルを設定します。このレベル未満のログはキューに送信されません。

### 一般的なメソッド

#### タスク実行 (Executor)
-   `start_executor(...)`: エグゼキューターの起動を記録します。
-   `end_executor(...)`: エグゼキューターの終了を記録し、所要時間と統計情報を含みます。

#### ステージとレイヤー (Stage & Layer)
-   `start_stage` / `end_stage`: ステージのライフサイクルを記録します。
-   `start_layer` / `end_layer`: 階層スケジューリングにおける各レイヤーの実行詳細を記録します。

#### タスクライフサイクル (Task)
-   `task_input`: タスクが入力キューに入ります。
-   `task_success`: タスクが正常に完了し、結果のサマリーと所要時間を記録します。
-   `task_retry`: タスクが失敗しましたがリトライをトリガーします。
-   `task_error`: タスクが失敗しリトライ不可（最終的な失敗）。
-   `task_duplicate`: 重複タスクが検出されました。

#### タスクフロー (Flow)
-   `split_trace` / `split_success`: タスク分割プロセスを記録します。
-   `route_success`: タスクルーティング結果を記録します。
-   `put_item` / `get_item`: キューの低レベル操作を記録します（TRACE レベル）。

#### レポーター (Reporter)
-   `push_*_failed`: Web サーバーへのデータプッシュ失敗の警告を記録します。
-   `pull_*_failed`: Web サーバーからの設定プル失敗の警告を記録します。

汎用的な `info()` や `debug()` ではなくこれらの専用メソッドを使用することで、生成されるログが読みやすく、機械解析しやすくなります。
