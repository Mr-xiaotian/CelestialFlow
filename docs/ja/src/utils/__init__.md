# Utils モジュール

Utils モジュールは、CelestialFlow の汎用ユーティリティ関数およびヘルパークラスを提供します。パフォーマンスベンチマーク、データクローン、コレクション操作、デバッグツール、フォーマット機能を含みます。これらのツールは他のモジュールで広く使用されており、コードの再利用性と保守性を向上させます。

## モジュール概要

Utils モジュールには、一連の独立した実用的なツールが含まれており、各ツールは特定の汎用的な問題の解決に特化しています。これらのツールはステートレス、テスト可能、かつ組み合わせ可能に設計されており、プロジェクトのどこでも使用できます。

## ファイル説明

### パフォーマンスツール

1. **util_benchmark.py**
   - **役割**: 異なる実行モード間のパフォーマンス差を比較するためのベンチマークツール
   - **主要機能**:
     - 実行時間の計測と統計
     - メモリ使用量の監視
     - 並行パフォーマンステスト
     - パフォーマンス比較レポートの生成
   - **使用シーン**: 
     - タスク実行モードの選択最適化
     - パフォーマンスボトルネックの発見
     - 並列化効果の検証
     - キャパシティプランニングとリソース配分

### データ処理ツール

2. **util_clone.py**
   - **役割**: 複雑なオブジェクトの完全なコピーをサポートするディープクローンツール
   - **主要機能**:
     - 再帰的オブジェクトクローン
     - 循環参照の処理
     - カスタムクローン戦略
     - パフォーマンス最適化（キャッシュ、遅延クローン）
   - **対応型**: プリミティブ型、リスト、辞書、セット、カスタムオブジェクト、ネスト構造
   - **使用シーン**: タスクデータのコピー、状態スナップショット、テストデータの準備

3. **util_collections.py**
   - **役割**: 特定のデータ処理機能を提供するコレクション操作ツール
   - **主要関数**: `cluster_by_value_sorted()`: 値に基づいて辞書をクラスタリングしソート
   - **主要機能**:
     - 辞書エントリを値でグループ化
     - グループ化結果のソート
     - 値でソートされたクラスタリング結果を返却
   - **使用シーン**: データ分析、結果のグループ化、統計集計、パフォーマンス指標のクラスタリング

### 開発・デバッグツール

4. **util_debug.py**
   - **役割**: シリアライズの問題を特定するためのデバッグツール
   - **主要関数**: `find_unpickleable()`: オブジェクト内で pickle シリアライズできない部分を検出
   - **主要機能**:
     - オブジェクトのシリアライズ可能性を再帰的にチェック
     - pickle 失敗の原因となる具体的な属性を特定
     - 詳細なエラー情報を提供
     - マルチプロセス環境でのシリアライズ問題のデバッグを支援
   - **使用シーン**: マルチプロセスタスク実行、オブジェクトシリアライズのデバッグ、プロセス間通信の問題調査

5. **util_format.py**
   - **役割**: 可読性を向上させるデータフォーマットおよび表示ツール
   - **主要関数**:
     - `format_repr()`: オブジェクトの文字列表現を最大長制限付きでフォーマット
     - `format_table()`: 配置と罫線付きでテーブルデータをフォーマット
     - `format_duration()`: 時間間隔をフォーマット（秒を可読形式に変換）
     - `format_timestamp()`: タイムスタンプをフォーマット
     - `format_avg_time()`: 平均処理時間をフォーマット
   - **主要機能**: データの美化、テーブル生成、時間フォーマット、パフォーマンス指標の表示
   - **使用シーン**: ログ出力、パフォーマンスレポート、ベンチマーク結果の表示、デバッグ情報のフォーマット

## モジュール連携

### 内部連携
- すべてのツールは独立しており、単独で使用できます
- 一部のツール間には依存関係があります（例: `util_debug` が出力に `util_format` を使用する場合があります）
- ツールは統一されたインターフェース規約に従います

### 外部連携
- **全モジュールとの連携**: Utils モジュールは他のすべてのモジュールで広く使用されています
- **Runtime モジュールとの連携**: パフォーマンステストツールがエグゼキュータの性能テストに使用されます
- **Stage モジュールとの連携**: データクローンツールがタスク状態のコピーに使用されます
- **Graph モジュールとの連携**: コレクションツールがグラフデータ操作に使用されます
- **Persistence モジュールとの連携**: フォーマットツールがデータシリアライズの表示に使用されます

## 設計原則

### 単一責任
- 各ツールファイルは1種類の問題のみを解決します
- 関数は小さく専門的に設計し、機能の肥大化を避けます
- 明確なインターフェースと明確な責務

### ステートレス設計
- ツール関数は可能な限りステートレスです
- グローバル変数と副作用を回避します
- 並行安全な使用をサポートします

### テスト容易性
- ユニットテストが容易な純粋関数設計
- 依存性注入のサポート
- モックとスタブのサポート

### 拡張性
- カスタム拡張をサポートするプラグインアーキテクチャ
- 設定駆動で柔軟な動作調整
- バージョン互換性の保証

## 使用パターン

### 直接使用
```python
from celestialflow.utils import util_benchmark, util_clone, util_collections

# パフォーマンステスト
results = util_benchmark.run_benchmark(task_executor, iterations=100)

# ディープクローン
cloned_data = util_clone.deep_clone(complex_object)

# コレクション操作
grouped = util_collections.cluster_by_value_sorted(items)
```

### 統合使用
- タスクエグゼキュータにパフォーマンス監視を統合
- データパイプラインでコレクションツールを使用
- デバッグモードで診断ツールを有効化
- レポート生成でフォーマットツールを使用

### カスタム拡張
- 基本ツールクラスを継承して特定の機能を実装
- カスタムフォーマッタを追加
- ドメイン固有のコレクション操作を実装

## ベストプラクティス

1. **パフォーマンスクリティカルパス**: ホットパスでは重いツール（ディープクローンなど）の使用を避けてください
2. **エラー処理**: ツール関数は明確なエラーメッセージとリカバリオプションを提供すべきです
3. **リソース管理**: リソースを必要とするツール（ファイルハンドル、ネットワーク接続など）は適切にクリーンアップすべきです
4. **ドキュメントと例**: 複雑なツールには使用例と一般的なシナリオを提供してください
5. **バージョン互換性**: ツールインターフェースの後方互換性を維持し、破壊的変更を避けてください
6. **テストカバレッジ**: ツール関数に包括的なテストカバレッジ（特にエッジケース）を確保してください
