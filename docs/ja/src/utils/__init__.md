# Utils モジュール

> 📅 最終更新日: 2026/04/22

Utils モジュールは、CelestialFlow の汎用ユーティリティ関数とヘルパークラスを提供します。パフォーマンステスト、データクローン、コレクション操作、デバッグツール、フォーマット機能が含まれます。これらのツールは他のモジュールで広く使用され、コードの再利用性と保守性を向上させます。

## モジュール概要

Utils モジュールは、独立しつつも実用的な一連のツールを含み、それぞれが特定の汎用問題の解決に焦点を当てています。これらのツールはステートレス、テスト可能、組み合わせ可能に設計されており、プロジェクトのどこでも使用できます。

## ファイル説明

### パフォーマンスツール

1. **util_benchmark.py**
   - **役割**: 異なる実行モード間のパフォーマンス差を比較するためのパフォーマンスベンチマークツール
   - **主要機能**:
     - 実行時間の測定と統計
     - メモリ使用量の監視
     - 並行パフォーマンステスト
     - パフォーマンス比較レポートの生成
   - **使用シナリオ**: 
     - 実行モード選択の最適化
     - パフォーマンスボトルネックの発見
     - 並列化効果の検証
     - キャパシティプランニングとリソース割り当て

### データ処理ツール

2. **util_clone.py**
   - **役割**: 複雑なオブジェクトの完全なコピーをサポートするディープクローンツール
   - **主要機能**:
     - 再帰的オブジェクトクローン
     - 循環参照の処理
     - カスタムクローン戦略
     - パフォーマンス最適化（キャッシュ、遅延クローン）
   - **サポート型**: 基本型、リスト、辞書、セット、カスタムオブジェクト、ネスト構造
   - **使用シナリオ**: タスクデータのコピー、状態スナップショット、テストデータの準備

3. **util_collections.py**
   - **役割**: 特定のデータ処理機能を提供するコレクション操作ツール
   - **主要関数**: `cluster_by_value_sorted()`：値に基づいて辞書をクラスタリングおよびソート
   - **主要機能**:
     - 辞書を値でグループ化
     - グループ化結果のソート
     - 値でソートされたクラスタリング結果の返却
   - **使用シナリオ**: データ分析、結果グルーピング、統計サマリー、パフォーマンスメトリクスクラスタリング

### 開発・デバッグツール

4. **util_debug.py**
   - **役割**: シリアライゼーション問題の特定を支援するデバッグツール
   - **主要関数**: `find_unpickleable()`：オブジェクト内で pickle シリアライゼーションできない部分を検出
   - **主要機能**:
     - オブジェクトのシリアライゼーション可能性を再帰的にチェック
     - pickle 失敗の原因となる具体的な属性を特定
     - 詳細なエラー情報の提供
     - マルチプロセス環境でのシリアライゼーション問題のデバッグを支援
   - **使用シナリオ**: マルチプロセスタスク実行、オブジェクトシリアライゼーションデバッグ、クロスプロセス通信のトラブルシューティング

5. **util_format.py**
   - **役割**: 可読性を向上させるデータフォーマットおよび表示ツール
   - **主要関数**:
     - `format_repr()`：最大長制限付きでオブジェクトの文字列表現をフォーマット
     - `format_table()`：配置と枠線付きで表形式データをフォーマット
     - `format_duration()`：時間間隔をフォーマット（秒を可読形式に変換）
     - `format_timestamp()`：タイムスタンプをフォーマット
     - `format_avg_time()`：平均処理時間をフォーマット
   - **主要機能**: データ整形、テーブル生成、時間フォーマット、パフォーマンスメトリクス表示
   - **使用シナリオ**: ログ出力、パフォーマンスレポート、ベンチマーク結果表示、デバッグ情報フォーマット

## モジュール関連

### 内部関連
- すべてのツールは独立しており、単独で使用可能
- 一部のツール間に依存関係あり（例：`util_debug` が出力に `util_format` を使用する場合がある）
- ツール設計は統一インターフェース規約に従う

### 外部関連
- **すべてのモジュールとの関連**: Utils モジュールは他のすべてのモジュールで広く使用
- **Runtime モジュールとの関連**: パフォーマンステストツールでエグゼキューターの性能をテスト
- **Stage モジュールとの関連**: データクローンツールでタスク状態のコピー
- **Graph モジュールとの関連**: コレクションツールでグラフデータ操作
- **Persistence モジュールとの関連**: フォーマットツールでデータシリアライゼーション表示

## 設計原則

### 単一責任
- 各ツールファイルは1カテゴリの問題のみを解決
- 関数は小さく焦点を絞って設計、機能肥大化を回避
- 明確なインターフェースと明確な責任

### ステートレス設計
- ユーティリティ関数は可能な限りステートレス
- グローバル変数と副作用を回避
- 並行安全な使用をサポート

### テスト可能性
- 純粋関数設計で容易なユニットテスト
- 依存性注入のサポート
- モックとスタブのサポート

### 拡張性
- カスタム拡張をサポートするプラグインベースのアーキテクチャ
- 設定駆動、柔軟な動作調整
- バージョン互換性の保証

## 使用パターン

### 直接使用
```python
from celestialflow.utils import util_benchmark, util_clone, util_collections

# パフォーマンステスト
results = util_benchmark.run_benchmark(task_executor, iterations=100)

# ディープクローン
cloned_data = util_clone.deep_clone(complex_object)

# コレクション操作
grouped = util_collections.cluster_by_value_sorted(items)
```

### 統合使用
- タスクエグゼキューターにパフォーマンス監視を統合
- データパイプラインでコレクションツールを使用
- デバッグモードで診断ツールを有効化
- レポート生成でフォーマットツールを使用

### カスタム拡張
- 基底ツールクラスを継承して特定機能を実装
- カスタムフォーマッターの追加
- ドメイン固有のコレクション操作の実装

## ベストプラクティス

1. **パフォーマンスクリティカルパス**: ホットパスでの重いツール（ディープクローンなど）の使用を回避
2. **エラーハンドリング**: ツール関数は明確なエラーメッセージとリカバリオプションを提供すべき
3. **リソース管理**: リソースを必要とするツール（ファイルハンドル、ネットワーク接続）は適切にクリーンアップ
4. **ドキュメントと例**: 複雑なツールには使用例と一般的なシナリオを提供
5. **バージョン互換性**: ツールインターフェースの後方互換性を維持、破壊的変更を回避
6. **テストカバレッジ**: ツール関数の完全なテストカバレッジを確保、特に境界ケース
