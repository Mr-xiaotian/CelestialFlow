# Web 模块

Web 模块提供了 CelestialFlow 的 Web 监控和管理界面，基于 FastAPI 构建，支持实时任务状态展示、错误监控、任务注入和远程控制功能。

## 模块概述

Web 模块是一个独立的 Web 服务器，为 CelestialFlow 提供可视化的监控和管理界面。它接收来自 `TaskReporter` 的状态数据，提供实时监控视图，并支持通过 Web 界面进行任务注入和系统控制。

## 文件说明

### 核心组件

1. **server.py** (`TaskWebServer`)
   - **作用**: 基于 FastAPI 的 Web 服务器，提供 RESTful API 和 WebSocket 接口
   - **关键功能**:
     - **状态展示**: 实时显示任务图结构、节点状态、性能指标
     - **错误监控**: 展示错误详情、堆栈跟踪、发生时间线
     - **任务注入**: 通过 Web 界面动态注入新任务到运行中的任务图
     - **远程控制**: 支持暂停、恢复、重启、终止等控制操作
     - **历史分析**: 查看历史执行记录和性能趋势
   - **技术栈**: FastAPI、WebSocket、Jinja2 模板、静态文件服务

## 模块关联

### 内部关联
- `TaskWeb` 是 Web 模块的唯一核心组件
- 包含前端静态文件（HTML、CSS、JavaScript）和后端 API 服务

### 外部关联
- **与 Observability 模块**: 接收 `TaskReporter` 上报的状态数据
- **与 Graph 模块**: 展示任务图结构和节点状态
- **与 Runtime 模块**: 展示执行状态和性能指标
- **与 Stage 模块**: 展示任务节点详情和执行结果
- **与 Persistence 模块**: 访问持久化的日志和错误数据

## 架构特点

### 前后端分离
- **后端**: FastAPI 提供 RESTful API 和 WebSocket 服务
- **前端**: 静态 HTML/JS/CSS 文件，支持现代浏览器
- **通信**: JSON over HTTP/WebSocket，支持实时更新

### 实时性支持
- **WebSocket**: 实时状态推送，减少轮询开销
- **Server-Sent Events**: 可选的数据流推送方式
- **长轮询**: 兼容不支持 WebSocket 的环境

### 可扩展性
- **插件架构**: 支持自定义监控面板和控件
- **主题系统**: 支持界面主题切换和自定义
- **多语言**: 支持国际化界面
- **权限系统**: 基于角色的访问控制

### 安全性
- **CORS 配置**: 控制跨域访问
- **认证授权**: 支持多种认证方式（API Key、OAuth、JWT）
- **输入验证**: 严格的输入数据验证和清理
- **速率限制**: 防止 API 滥用

## 功能特性

### 监控面板
- **拓扑视图**: 可视化展示任务图结构和依赖关系
- **状态面板**: 实时显示节点状态（颜色编码）
- **性能仪表盘**: 显示 CPU、内存、队列长度等指标
- **错误中心**: 集中展示错误信息和统计

### 控制功能
- **任务注入**: 通过表单或 JSON 定义新任务
- **参数调整**: 动态调整系统参数和配置
- **执行控制**: 启动、暂停、恢复、终止任务执行
- **数据导出**: 导出执行日志、性能数据、错误报告

### 分析功能
- **历史趋势**: 查看性能指标的历史变化趋势
- **对比分析**: 对比不同时间段的执行情况
- **根因分析**: 错误根因分析和关联性分析
- **报表生成**: 自动生成执行报告和性能分析报告

## 使用模式

### 启动服务器
```bash
# 命令行启动
celestialflow-web

# 指定端口和主机
celestialflow-web --host 0.0.0.0 --port 5080
```

### API 使用
```python
import requests

# 获取任务图状态
response = requests.get("http://localhost:5000/api/get_status")
status = response.json()

# 注入新任务（先 POST 到 push_injection_tasks，再由 TaskGraph 拉取）
task_injection = {
    "node": "Stage 1",
    "task_datas": [[1, 2, 3]],
    "timestamp": "2024-01-01T00:00:00"
}
response = requests.post("http://localhost:5000/api/push_injection_tasks", json=task_injection)
```

### WebSocket 连接
```javascript
// 前端 JavaScript 示例
const ws = new WebSocket("ws://localhost:5000/ws/status");

ws.onmessage = (event) => {
    const data = JSON.parse(event.data);
    // 更新界面
    updateStatus(data);
};

// 发送控制命令
ws.send(JSON.stringify({
    action: "subscribe",
    channel: "metrics"
}));
```
