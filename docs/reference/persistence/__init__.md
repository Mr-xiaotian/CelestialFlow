# Persistence 模块

Persistence 模块提供了 CelestialFlow 的数据持久化功能，包括执行日志记录、错误信息存储、任务状态保存和配置常量管理。它确保任务执行的关键数据能够可靠地保存和检索。

## 模块概述

Persistence 模块负责将运行时数据持久化到各种存储后端，支持日志记录、错误追踪、状态快照和配置管理。模块采用出口模式，可以无缝集成到任务执行流程中，不影响主流程性能。

## 文件说明

### 基础架构

1. **core_funnel.py** (`BaseSpout`, `BaseInlet`)
   - **作用**: 持久化系统的基础架构，提供多进程安全的队列通信和后台监听
   - **核心组件**:
     - `BaseSpout`: 所有出口类的基类，提供通用监听功能
     - `BaseInlet`: 所有收集器类的基类，提供通用收集功能
   - **关键功能**: 多进程安全、后台线程、优雅启停、流量控制

### 日志持久化

2. **core_log.py** (`LogSpout`, `LogInlet`)
   - **作用**: 日志记录和存储的基础架构
   - **核心组件**:
     - `LogSpout`: 日志出口，从队列接收日志消息并写入文件
     - `LogInlet`: 日志收集器，将日志写入文件
   - **日志格式**: 文本日志格式，包含时间戳、日志级别、消息内容
   - **关键功能**: 异步写入、文件管理、日志轮转、多进程安全

### 错误持久化

3. **core_fail.py** (`FailSpout`, `FailInlet`)
   - **作用**: 错误信息记录和存储的基础架构
   - **核心组件**:
     - `FailSpout`: 错误出口，从队列接收错误信息并写入文件
     - `FailInlet`: 错误收集器，将错误信息写入文件
   - **错误信息**: 错误类型、堆栈跟踪、上下文数据、发生时间、任务ID
   - **关键功能**: 错误分类、文件存储、异步处理、多进程安全

### 数据格式和配置

4. **util_jsonl.py**
   - **作用**: JSON Lines 格式支持，用于高效的结构化数据存储和读取
   - **关键功能**:
     - `load_jsonl_logs()`: 从 JSONL 文件加载日志数据，支持选择性字段读取
     - 流式数据读取，支持从指定行号开始
     - 数据过滤和转换
     - 错误处理和文件验证
   - **使用场景**: 日志文件读取、错误记录分析、Web 界面数据展示

5. **util_constant.py**
   - **作用**: 持久化相关的常量和配置定义
   - **包含内容**:
     - `LEVEL_DICT`: 日志级别字典，定义日志级别和对应的数值
     - 日志级别包括: TRACE(0), DEBUG(10), SUCCESS(20), INFO(30), WARNING(40), ERROR(50), CRITICAL(60)
   - **关键功能**: 统一的日志级别管理、级别比较、日志过滤

## 模块关联

### 内部关联
- 所有持久化类都继承自 `BaseSpout`/`BaseInlet`
- `LogSpout`/`LogInlet` 和 `FailSpout`/`FailInlet` 配对使用
- 工具类被核心类使用，提供格式支持和配置管理

### 外部关联
- **与 Runtime 模块**: 监听运行时产生的日志和错误
- **与 Stage 模块**: 记录任务执行状态和结果
- **与 Graph 模块**: 保存图结构状态和执行历史
- **与 Observability 模块**: 提供原始数据用于监控和分析
- **与 Utils 模块**: 使用工具函数进行数据处理和格式化

## 架构特点

### 异步非阻塞设计
- 出口在后台线程运行，不阻塞主流程
- 队列缓冲，应对写入峰值
- 批量提交，提高存储效率

### 多后端支持
- 支持文件、数据库、远程服务等多种存储
- 统一的接口，便于切换和扩展
- 存储适配器模式，支持自定义后端

### 可靠性保证
- 写入失败重试机制
- 数据完整性校验
- 存储空间监控和告警
- 优雅降级，主流程不受影响

### 可配置性
- 存储路径可配置
- 格式和编码可调整
- 性能参数可调优
- 告警规则可自定义

## 使用模式

### 基础配置
```python
from celestialflow.persistence import LogSpout, LogInlet, FailSpout, FailInlet

# 配置日志持久化
log_spout = LogSpout()
log_spout.start()
log_inlet = LogInlet(log_spout.get_queue())

# 配置错误持久化
fail_spout = FailSpout(error_source="graph_errors")
fail_spout.start()
fail_inlet = FailInlet(fail_spout.get_queue())
```

### 高级使用
1. **多存储后端**: 同时写入文件和数据库，实现冗余存储
2. **自定义格式**: 实现自定义的序列化格式
3. **实时告警**: 配置错误阈值，触发实时通知
4. **数据归档**: 自动归档历史数据，释放存储空间
5. **数据迁移**: 在不同存储后端间迁移数据

### 监控和维护
1. **存储监控**: 监控磁盘使用率，预防存储溢出
2. **性能监控**: 监控写入延迟和吞吐量
3. **错误监控**: 监控持久化失败率
4. **数据清理**: 定期清理过期数据

## 最佳实践

### 生产环境配置
1. **存储规划**: 根据数据量和保留策略规划存储空间
2. **冗余设计**: 重要数据使用多副本存储
3. **性能优化**: 根据负载调整批处理大小和并发度
4. **监控告警**: 配置完整的监控和告警规则

### 开发和测试
1. **本地存储**: 开发环境使用文件存储，便于调试
2. **模拟后端**: 测试环境使用内存或模拟后端
3. **数据隔离**: 不同环境使用不同的存储路径
4. **清理策略**: 测试数据自动清理，避免积累

### 故障处理
1. **写入失败**: 配置适当的重试策略和降级方案

4. **格式错误**: 数据验证和格式兼容性处理
