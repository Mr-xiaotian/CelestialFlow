# Utils 模块

Utils 模块提供了 CelestialFlow 的通用工具函数和辅助类，包括性能测试、数据克隆、集合操作、调试工具和格式化功能。这些工具被其他模块广泛使用，提高了代码的复用性和可维护性。

## 模块概述

Utils 模块包含了一系列独立但实用的工具，每个工具都专注于解决特定的通用问题。这些工具设计为无状态、可测试和可组合的，可以在项目的任何地方使用。

## 文件说明

### 性能工具

1. **util_benchmark.py**
   - **作用**: 性能基准测试工具，用于对比不同执行模式的性能差异
   - **关键功能**:
     - 执行时间测量和统计
     - 内存使用监控
     - 并发性能测试
     - 性能对比报告生成
   - **使用场景**: 
     - 优化任务执行模式选择
     - 发现性能瓶颈
     - 验证并行化效果
     - 容量规划和资源分配

### 数据处理工具

2. **util_clone.py**
   - **作用**: 深度克隆工具，支持复杂对象的完整复制
   - **关键功能**:
     - 递归对象克隆
     - 循环引用处理
     - 自定义克隆策略
     - 性能优化（缓存、懒克隆）
   - **支持类型**: 基本类型、列表、字典、集合、自定义对象、嵌套结构
   - **使用场景**: 任务数据复制、状态快照、测试数据准备

3. **util_collections.py**
   - **作用**: 集合操作工具，提供特定的数据处理功能
   - **关键函数**: `cluster_by_value_sorted()`: 根据值对字典进行聚类和排序
   - **关键功能**:
     - 将字典按值分组
     - 对分组结果进行排序
     - 返回按值排序的聚类结果
   - **使用场景**: 数据分析、结果分组、统计汇总、性能指标聚类

### 开发和调试工具

4. **util_debug.py**
   - **作用**: 调试工具，帮助识别序列化问题
   - **关键函数**: `find_unpickleable()`: 查找对象中无法被 pickle 序列化的部分
   - **关键功能**:
     - 递归检查对象的可序列化性
     - 识别导致 pickle 失败的具体属性
     - 提供详细的错误信息
     - 帮助调试多进程环境下的序列化问题
   - **使用场景**: 多进程任务执行、对象序列化调试、跨进程通信问题排查

5. **util_format.py**
   - **作用**: 数据格式化和展示工具，提高可读性
   - **关键函数**:
     - `format_repr()`: 格式化对象的字符串表示，限制最大长度
     - `format_table()`: 格式化表格数据，支持对齐和边框
     - `format_duration()`: 格式化时间间隔（秒转换为可读格式）
     - `format_timestamp()`: 格式化时间戳
     - `format_avg_time()`: 格式化平均处理时间
   - **关键功能**: 数据美化、表格生成、时间格式化、性能指标展示
   - **使用场景**: 日志输出、性能报告、基准测试结果展示、调试信息格式化

## 模块关联

### 内部关联
- 所有工具都是独立的，可以单独使用
- 部分工具之间存在依赖关系（如 `util_debug` 可能使用 `util_format` 进行输出）
- 工具设计遵循统一的接口规范

### 外部关联
- **与所有模块**: Utils 模块被其他所有模块广泛使用
- **与 Runtime 模块**: 性能测试工具用于测试执行器性能
- **与 Stage 模块**: 数据克隆工具用于任务状态复制
- **与 Graph 模块**: 集合工具用于图数据操作
- **与 Persistence 模块**: 格式化工具用于数据序列化展示

## 设计原则

### 单一职责
- 每个工具文件只解决一类问题
- 函数设计小而专，避免功能膨胀
- 清晰的接口和明确的职责

### 无状态设计
- 工具函数尽可能无状态
- 避免全局变量和副作用
- 支持并发安全使用

### 可测试性
- 纯函数设计，便于单元测试
- 依赖注入支持
- 模拟和桩支持

### 可扩展性
- 插件式架构，支持自定义扩展
- 配置驱动，灵活调整行为
- 版本兼容性保证

## 使用模式

### 直接使用
```python
from celestialflow.utils import util_benchmark, util_clone, util_collections

# 性能测试
results = util_benchmark.run_benchmark(task_executor, iterations=100)

# 深度克隆
cloned_data = util_clone.deep_clone(complex_object)

# 集合操作
grouped = util_collections.cluster_by_value_sorted(items)
```

### 集成使用
- 在任务执行器中集成性能监控
- 在数据流水线中使用集合工具
- 在调试模式启用诊断工具
- 在报告生成中使用格式化工具

### 自定义扩展
- 继承基础工具类实现特定功能
- 添加自定义的格式化器
- 实现领域特定的集合操作

## 最佳实践

1. **性能关键路径**: 避免在热路径中使用重型工具（如深度克隆）
2. **错误处理**: 工具函数应提供清晰的错误信息和恢复选项
3. **资源管理**: 需要资源的工具（如文件句柄、网络连接）应正确清理
4. **文档和示例**: 为复杂工具提供使用示例和常见场景
5. **版本兼容**: 工具接口保持向后兼容，避免破坏性变更
6. **测试覆盖**: 确保工具函数有完整的测试覆盖，特别是边界情况
